# Claude Skills 迭代管理器

## 规范来源

> 本 Skill 基于 Claude Skills 使用规范实现

## 触发条件

- 月度 AI 使用复盘后需要更新 Skills
- feedback-loop-collector 输出 Skills 更新建议
- 开发者反馈技能缺失或效果不佳
- 新规范发布需要新增对应 Skill
- Skills 版本升级评审

## 工作流

### Phase 1: 问题收集

从多个来源收集 Skills 相关问题：

```
问题来源
    │
    ├── feedback-loop-collector
    │   ├── Skills 使用频率统计
    │   ├── Skills 效果评分
    │   └── Skills 优化建议
    │
    ├── 开发者反馈
    │   ├── 技能缺失需求
    │   ├── 技能效果不佳
    │   ├── 技能误触发/漏触发
    │   └── 技能交互体验
    │
    ├── 规范变更
    │   ├── 新规范发布 → 需要新增 Skill
    │   ├── 规范修订 → 需要更新 Skill
    │   └── 规范废弃 → 需要废弃 Skill
    │
    └── 技术演进
        ├── 新技术栈引入 → 需要新增 Skill
        ├── 工具链升级 → 需要更新 Skill
        └── 最佳实践变化 → 需要更新 Skill
```

### Phase 2: 月度评审

对收集的问题进行月度评审：

```
月度评审流程
    │
    ├── 1. 使用数据分析
    │   ├── 各 Skill 使用频率排名
    │   ├── 各 Skill 效果评分排名
    │   ├── 零使用 Skill 识别
    │   └── 高频低效 Skill 识别
    │
    ├── 2. 问题分类
    │   ├── 🆕 新增需求 → 评估优先级
    │   ├── 🔧 更新需求 → 评估影响范围
    │   ├── 🗑️ 废弃需求 → 评估迁移成本
    │   └── 🔄 重构需求 → 评估投入产出
    │
    ├── 3. 优先级排序
    │   ├── P0：安全/合规相关 → 立即执行
    │   ├── P1：高频使用/高影响 → 本月执行
    │   ├── P2：中频使用/中影响 → 下月执行
    │   └── P3：低频使用/低影响 → 观望
    │
    └── 4. 资源评估
        ├── 预估工时
        ├── 依赖关系
        └── 风险评估
```

### Phase 3: 技能更新

执行 Skills 的更新操作：

```
技能更新类型
    │
    ├── 🆕 新增 Skill
    │   ├── 创建目录结构
    │   ├── 编写 SKILL.md
    │   ├── 编写检查清单/模板
    │   ├── 更新 CLAUDE.md 和 README.md
    │   └── 团队评审
    │
    ├── 🔧 更新 Skill
    │   ├── 修改 SKILL.md 内容
    │   ├── 更新检查清单/模板
    │   ├── 递增版本号
    │   └── 更新 README.md
    │
    ├── 🗑️ 废弃 Skill
    │   ├── 迁移到 legacy/ 目录
    │   ├── 更新 README.md 标注替代方案
    │   ├── 通知团队成员
    │   └── 过渡期后删除
    │
    └── 🔄 重构 Skill
        ├── 合并相似 Skill
        ├── 拆分过大 Skill
        ├── 调整目录结构
        └── 更新关联 Skill
```

### Phase 4: 验证发布

更新后的 Skills 需要验证才能发布：

```
验证流程
    │
    ├── 1. 语法验证
    │   ├── SKILL.md 格式正确
    │   ├── frontmatter 字段完整
    │   └── 引用路径有效
    │
    ├── 2. 功能验证
    │   ├── 触发条件是否正确
    │   ├── 工作流是否完整
    │   └── 输入/输出是否符合预期
    │
    ├── 3. 集成验证
    │   ├── 与关联 Skill 是否兼容
    │   ├── 与全局规则是否一致
    │   └── 与项目规范是否对齐
    │
    └── 4. 发布
        ├── 更新版本号
        ├── 更新 CLAUDE.md + README.md
        ├── 提交 Git 变更
        └── 通知团队成员
```

### Phase 5: 输出报告

```markdown
## 🔄 Skills 迭代报告

**评审周期**：YYYY-MM
**评审日期**：YYYY-MM-DD

### 1. Skills 使用概览

| Skill | 使用次数 | 效果评分 | 状态 |
|-------|----------|----------|------|
| vibe-verify | 45 | ⭐⭐⭐⭐ | 正常 |
| code-review-expert | 32 | ⭐⭐⭐⭐⭐ | 正常 |
| deprecated-skill | 0 | — | 🗑️ 建议废弃 |

### 2. 本月变更

| 类型 | Skill | 变更内容 | 版本 |
|------|-------|----------|------|
| 🔧 更新 | code-review-expert | 对齐  + AI 专项 | v1.0 → v2.0 |
| 🆕 新增 | fix-priority-triage | 修复优先级分诊 | v1.0 |
| 🗑️ 废弃 | old-review-skill | 迁移到 legacy/ | — |

### 3. 下月计划

| 优先级 | 类型 | Skill | 说明 |
|--------|------|-------|------|
| P1 | 🔧 更新 | vibe-verify | 增加自动触发 |
| P2 | 🆕 新增 | frontend-code-gen | 前端代码生成 |
| P3 | 🗑️ 废弃 | legacy-skill | 过渡期结束，删除 |

### 4. 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| [风险 1] | [影响] | [措施] |
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 操作类型 | 枚举 | 否 | collect（收集）/ review（评审）/ update（更新）/ verify（验证）/ report（报告），默认 report |
| 反馈数据 | 文件引用 | 否 | feedback-loop-collector 的输出 |
| 变更内容 | 对象 | 否 | 更新时的具体变更 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 使用概览 | 表格 | 各 Skill 使用频率和效果评分 |
| 变更清单 | 表格 | 本月 Skills 变更记录 |
| 下月计划 | 表格 | 下月 Skills 迭代计划 |
| 风险评估 | 表格 | 变更风险和缓解措施 |

## 检查清单

- [ ] 是否收集了所有反馈来源的问题？
- [ ] 是否按优先级排序了变更需求？
- [ ] 新增 Skill 是否有完整的 SKILL.md？
- [ ] 更新 Skill 是否递增了版本号？
- [ ] 废弃 Skill 是否迁移到 legacy/？
- [ ] 是否更新了 CLAUDE.md 和 README.md？
- [ ] 是否验证了更新后的 Skill 功能？
- [ ] 是否通知了团队成员？

## 与其他 Skills 的关系

| 关联 Skill | 关系类型 | 说明 |
|------------|----------|------|
| feedback-loop-collector | 上游 | 提供 Skills 使用效果反馈 |
| entropy-manager | 上游 | 提供代码健康度数据 |
| 所有 Skills | 管控 | 本 Skill 管理所有其他 Skills 的生命周期 |
