# Skills 迭代评审清单

## 1. 使用数据分析

| 检查项 | 说明 | 数据来源 |
|--------|------|----------|
| 使用频率 | 各 Skill 被触发/调用的次数 | feedback-loop-collector |
| 效果评分 | 用户对 Skill 效果的主观评分(1-5) | feedback-loop-collector |
| 零使用识别 | 连续 N 个月未被使用的 Skill | 使用统计 |
| 高频低效 | 使用频率高但效果评分低的 Skill | 交叉分析 |

## 2. 问题分类

| 类型 | 判定标准 | 处理方式 |
|------|----------|----------|
| 🆕 新增 | 缺失的能力需求 | 评估优先级后新增 |
| 🔧 更新 | 现有 Skill 需要优化 | 评估影响后更新 |
| 🗑️ 废弃 | 使用率极低或有替代 | 迁移到 legacy/ |
| 🔄 重构 | 结构/组织需要调整 | 评估成本后重构 |

## 3. 优先级排序

| 优先级 | 判定标准 | 执行时机 |
|--------|----------|----------|
| P0 | 安全/合规相关 | 立即执行 |
| P1 | 高频使用/高影响 | 本月执行 |
| P2 | 中频使用/中影响 | 下月执行 |
| P3 | 低频使用/低影响 | 观望 |

## 4. 新增 Skill 检查

| 检查项 | 说明 |
|--------|------|
| SKILL.md 格式 | frontmatter 完整、工作流清晰 |
| 检查清单/模板 | 配套的 checklists/ 或 templates/ |
| README.md 更新 | 技能总览中已列出 |
| CLAUDE.md 更新 | Skills Structure 中已列出 |
| 规范对齐 | 有明确的规范来源说明 |
| 关联 Skill | 与其他 Skills 的关系已声明 |

## 5. 更新 Skill 检查

| 检查项 | 说明 |
|--------|------|
| 版本号递增 | major/minor/patch 合理递增 |
| 变更说明 | SKILL.md 中标注了变更内容 |
| 向后兼容 | 不影响现有使用方式 |
| 关联更新 | 关联 Skill 是否需要同步更新 |
| README.md 同步 | 描述是否已更新 |

## 6. 废弃 Skill 检查

| 检查项 | 说明 |
|--------|------|
| 替代方案 | 是否有替代 Skill |
| 迁移路径 | 用户如何迁移到替代方案 |
| legacy/ 迁移 | 已迁移到 legacy/ 目录 |
| README.md 标注 | 已标注废弃和替代方案 |
| 通知团队 | 已通知团队成员 |
| 过渡期 | 设定了合理的过渡期 |

## 7. 验证发布

| 检查项 | 说明 |
|--------|------|
| 语法验证 | SKILL.md 格式正确 |
| 功能验证 | 触发/工作流/输出正常 |
| 集成验证 | 与关联 Skill 兼容 |
| 文档同步 | CLAUDE.md + README.md 已更新 |
| Git 提交 | 变更已提交 |
| 团队通知 | 已通知更新内容 |
