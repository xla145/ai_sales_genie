# 三级边界执行器

## 规范来源

> 本 Skill 基于 三级边界系统实现

## 触发条件

- AI 即将执行文件删除操作
- AI 即将修改配置文件或设计文档
- AI 即将引入新依赖或跨模块变更
- AI 即将执行 Git 提交或合并操作
- AI 即将处理敏感数据
- 用户询问某个操作是否被允许

## 工作流

### Phase 1: 操作分类

对即将执行的操作进行分类判定：

```
操作类型判断
    │
    ├── 文件操作
    │   ├── 删除临时文件/日志/IDE 生成文件 → ✅ Always（直接执行）
    │   ├── 删除 _tmp/_bak 文件 → ✅ Always（直接执行）
    │   ├── 删除源码文件 → ⚠️ Ask First
    │   ├── 删除配置文件 → ⚠️ Ask First
    │   └── 批量删除（≥2 个文件）→ ⚠️ Ask First
    │
    ├── 代码变更
    │   ├── 遵循编码规范 → ✅ Always
    │   ├── 保持 design.md 一致性 → ✅ Always
    │   ├── 补充注释和文档 → ✅ Always
    │   ├── 修改非当前任务的 design.md → ⚠️ Ask First
    │   ├── 修改 AGENTS.md/Skills 配置 → ⚠️ Ask First
    │   ├── 跨模块依赖变更 → ⚠️ Ask First
    │   └── 引入新外部依赖 → ⚠️ Ask First
    │
    ├── Git 操作
    │   ├── 运行代码检查和测试 → ✅ Always
    │   ├── 直接提交到 main/生产分支 → 🚫 Never
    │   └── 绕过代码审查直接合并 → 🚫 Never
    │
    └── 安全操作
        ├── 遵守安全编码规范 → ✅ Always
        ├── 暴露敏感信息 → 🚫 Never
        ├── 执行破坏性数据库操作 → 🚫 Never
        └── 绕过安全检查 → 🚫 Never
```

### Phase 2: 执行判定

根据分类结果执行：

| 边界级别 | 执行方式 | 输出格式 |
|----------|----------|----------|
| ✅ **Always** | 直接执行，无需确认 | （无额外输出，正常执行） |
| ⚠️ **Ask First** | 暂停，向用户说明原因并请求确认 | `⚠️ 需要确认：[操作描述]。原因：[为什么需要确认]。是否继续？` |
| 🚫 **Never** | 拒绝执行，说明原因并给出替代方案 | `🚫 禁止执行：[操作描述]。原因：[为什么禁止]。替代方案：[建议]` |

### Phase 3: 记录与追溯

对于 ⚠️ 和 🚫 的判定，记录到交互上下文中：

- 操作内容
- 判定结果
- 判定依据（规则条目）
- 用户决策（如为 Ask First）

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 操作描述 | 文本 | 是 | 即将执行的操作 |
| 操作类型 | 枚举 | 是 | file_delete / file_modify / code_change / git_op / security_op |
| 影响范围 | 文本 | 否 | 操作影响的文件/模块/系统 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 边界判定 | ✅/⚠️/🚫 | 操作的边界级别 |
| 判定依据 | 文本 | 对应的规则条目 |
| 用户确认 | 是/否/替代 | 仅 Ask First 级别需要 |
| 替代方案 | 文本 | 仅 Never 级别提供 |

## 检查清单

执行边界检查时确认：

- [ ] 操作是否涉及文件删除？是否为临时文件？
- [ ] 操作是否涉及配置或设计文档修改？是否为当前任务范围？
- [ ] 操作是否引入新依赖或跨模块变更？
- [ ] 操作是否涉及 Git 提交到受保护分支？
- [ ] 操作是否涉及敏感信息处理？
- [ ] 操作是否涉及数据库写操作？
