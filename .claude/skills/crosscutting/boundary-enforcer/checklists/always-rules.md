# 始终执行规则（Always Rules）

> 每次操作必须遵守，无需询问用户

## 代码规范

- 遵循项目编码规范和格式化要求
- 保持与 design.md 的一致性
- 补充必要的注释和文档
- 遵守安全编码规范
- 运行代码检查和测试

## 文件清理

以下文件可直接删除，无需确认：

| 文件类型 | 示例 | 说明 |
|----------|------|------|
| 临时文件 | `*.tmp`, `*.temp` | 明确的临时文件 |
| 日志文件 | `*.log` | 运行时日志 |
| IDE 生成文件 | `.class`, `.pyc` | 编译产物 |
| 备份文件 | `*_tmp.*`, `*_bak.*` | 文件名含 `_tmp` 或 `_bak` |
| 系统文件 | `.DS_Store` | 操作系统生成 |

## 质量保障

- 代码生成后执行自检清单
- 单元测试覆盖核心路径
- 异常处理逻辑完整
- 无硬编码魔法值
