# 文档同步检查器

## 规范来源

> 本 Skill 基于 文档同步原则实现

## 核心原则

> ⚠️ **关键原则**：`design.md` 是代码变更的唯一真相来源

## 触发条件

- 代码变更涉及接口定义修改
- 数据结构/实体类变更
- 新增或删除功能模块
- Bug 修复涉及设计逻辑变更
- 代码重构改变模块结构
- 功能下线

## 工作流

### Phase 1: 变更检测

检测代码变更是否需要同步文档：

```
代码变更检测
    │
    ├── 接口变更
    │   ├── 新增 API 端点 → 需要更新 design.md
    │   ├── 修改 API 签名 → 需要更新 design.md
    │   └── 删除 API 端点 → 需要更新 design.md
    │
    ├── 数据结构变更
    │   ├── 新增/修改 Entity 字段 → 需要更新 design.md
    │   ├── 修改 DTO 结构 → 需要更新 design.md
    │   └── 数据库表结构变更 → 需要更新 design.md
    │
    ├── 模块结构变更
    │   ├── 新增模块 → 需要更新 AGENTS.md + design.md
    │   ├── 模块重命名 → 需要更新 AGENTS.md + design.md
    │   └── 模块删除 → 需要更新 AGENTS.md + design.md
    │
    └── 其他变更
        ├── Bug 修复涉及设计逻辑 → 需要补充 design.md
        ├── 重构改变模块结构 → 需要更新 design.md
        └── 功能下线 → 需要标记 design.md
```

### Phase 2: 文档一致性检查

对比代码与文档的一致性：

```
一致性检查
    │
    ├── 接口一致性
    │   ├── 代码中的 API 端点 vs design.md 中的接口定义
    │   ├── 请求/响应参数 vs design.md 中的数据结构
    │   └── HTTP 方法/路径 vs design.md 中的路由定义
    │
    ├── 数据结构一致性
    │   ├── Entity 字段 vs design.md 中的数据模型
    │   ├── DTO 字段 vs design.md 中的传输对象
    │   └── 数据库表 vs design.md 中的表结构
    │
    ├── 模块结构一致性
    │   ├── 代码包结构 vs AGENTS.md 中的模块声明
    │   ├── 模块依赖关系 vs AGENTS.md 中的依赖图
    │   └── 技术栈版本 vs AGENTS.md 中的技术栈
    │
    └── 业务逻辑一致性
        ├── 代码实现 vs design.md 中的流程描述
        ├── 状态流转 vs design.md 中的状态机
        └── 校验规则 vs design.md 中的业务规则
```

### Phase 3: 文档同步操作

根据变更场景执行文档同步：

| 场景 | 操作 | 文档更新 | 时机 |
|------|------|----------|------|
| 新功能开发 | 创建 design.md | ✅ 开发前完成 | Propose 阶段 |
| 设计迭代 | 更新 design.md | ✅ 迭代后同步 | Design 阶段 |
| Bug 修复 | 补充 design.md | ✅ 修复后同步 | Verify 阶段 |
| 代码重构 | 更新 design.md | ✅ 重构后同步 | Build 阶段 |
| 功能下线 | 标记 design.md | ✅ 下线前完成 | Archive 阶段 |

### Phase 4: 文档版本控制

| 文档类型 | 版本号 | 更新触发 | 审核人 |
|----------|--------|----------|--------|
| AGENTS.md | 主版本号 | 架构变更 | 架构师 |
| design.md | 次版本号 | 设计迭代 | 技术负责人 |
| API 文档 | 修订号 | 接口变更 | 技术负责人 |
| 测试文档 | 修订号 | 测试变更 | 测试负责人 |

### Phase 5: 输出报告

```markdown
## 📝 文档同步检查报告

**检查范围**：[变更文件列表]
**检查结果**：[同步 ✅ / 需更新 ⚠️ / 严重脱节 🚫]

### 1. 需要同步的文档

| # | 文档 | 变更原因 | 同步操作 | 优先级 |
|---|------|----------|----------|--------|
| 1 | design.md (用户模块) | 新增登录接口 | 新增接口定义 | 🚫 必须立即 |
| 2 | AGENTS.md | 新增支付模块 | 新增模块声明 | ⚠️ 本迭代内 |

### 2. 一致性检查结果

| # | 检查项 | 代码 | 文档 | 状态 |
|---|--------|------|------|------|
| 1 | 用户登录接口 | POST /api/login | POST /api/auth | ⚠️ 不一致 |
| 2 | User Entity | name, phone, email | name, phone | ⚠️ 缺少 email |

### 3. 版本控制

| 文档 | 当前版本 | 建议版本 | 变更类型 |
|------|----------|----------|----------|
| design.md | v1.2 | v1.3 | 次版本号（设计迭代） |
| AGENTS.md | v2.0 | v3.0 | 主版本号（架构变更） |

### 4. 建议操作

1. [操作 1]
2. [操作 2]
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 变更文件 | 文件列表 | 是 | 需要检查的代码变更 |
| 检查模式 | 枚举 | 否 | full（完整检查）/ interface（仅接口）/ data（仅数据结构）/ module（仅模块），默认 full |
| 文档路径 | 文本 | 否 | design.md / AGENTS.md 路径，默认项目根目录 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 同步判定 | ✅/⚠️/🚫 | 同步/需更新/严重脱节 |
| 需同步文档 | 表格 | 需要更新的文档清单 |
| 一致性检查 | 表格 | 代码与文档的差异 |
| 版本控制 | 表格 | 文档版本建议 |
| 建议操作 | 列表 | 具体的同步操作 |

## 检查清单

- [ ] 代码变更是否涉及接口定义？
- [ ] 代码变更是否涉及数据结构？
- [ ] 代码变更是否涉及模块结构？
- [ ] design.md 是否与代码一致？
- [ ] AGENTS.md 是否与代码一致？
- [ ] API 文档是否与代码一致？
- [ ] 文档版本是否需要更新？
- [ ] 变更是否需要审核人确认？
