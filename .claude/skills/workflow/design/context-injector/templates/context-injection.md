# 上下文注入模板



## 标准上下文注入模板

```markdown
## 项目上下文

### 1. 项目概述

[项目名称] 是 [一句话描述项目目标]

### 2. 当前迭代目标

- 迭代版本：[如 v2.1]
- 迭代周期：[如 2026-04-15 ~ 2026-06-06]
- 本迭代重点：[核心交付目标描述]

### 3. 技术栈

- 后端：[技术栈]
- 前端：[技术栈]
- 数据库：[数据库类型]
- 部署：[部署方式]

### 4. 项目结构

[项目目录结构]

### 5. 核心模块

- [模块1]：[职责描述]
- [模块2]：[职责描述]

### 6. 架构约束

- 依赖方向：[如 Controller → Service → Repository]
- 禁止依赖：[如 跨模块直接调用]
- 分层规则：[如 按功能分包]

### 7. 当前任务

- 任务：[任务名称]
- 目标：[任务目标]
- 约束：[技术约束/业务约束]
- 参考文档：[design.md 链接]

### 8. 编码规范要点

- 命名规范：[关键规则]
- 代码风格：[关键规则]
- 安全规范：[关键规则]
```

## 上下文层级加载策略

### 团队级（AGENTS.md）

始终全量加载，包含：
- 项目概述
- 技术栈
- 项目结构
- 核心模块
- 架构约束
- 编码规范
- AI 使用约束

### 项目级（PROJECT.md）

如有则全量加载，包含：
- 项目里程碑
- 迭代计划
- 依赖版本

### 模块级（design.md）

按需加载当前任务相关模块，包含：
- 模块概述
- 接口定义
- 数据结构
- 核心逻辑
- 异常处理

### 任务级（TB 任务描述）

用户提供的任务信息，包含：
- 任务目标
- 验收标准
- 依赖关系
- 时间约束

## 上下文缺失处理

当关键上下文文件不存在时：

| 缺失文件 | 处理方式 | 提示语 |
|----------|----------|--------|
| AGENTS.md | 建议创建 | `⚠️ 项目缺少 AGENTS.md，建议创建以确保 AI 获得完整上下文。是否需要我帮你生成？` |
| design.md | 建议创建 | `⚠️ 当前模块缺少 design.md，建议创建以确保实现与设计一致。是否需要我帮你生成？` |
| PROJECT.md | 可选创建 | `ℹ️ 项目缺少 PROJECT.md，建议创建以记录迭代目标。` |
