# AGENTS.md 维护

## 规范来源

> 本 Skill 基于 AGENTS.md 规范实现

## 触发条件

- 项目初始化时创建 AGENTS.md
- 重大架构变更（模块增删、依赖关系变化）
- 技术栈升级（框架版本升级）
- 人员变动（角色职责调整）
- 月末 AI 使用复盘会后常规更新
- 用户要求"创建/更新 AGENTS.md"

## 工作流

### Phase 1: 现状分析

| 场景 | 操作 |
|------|------|
| AGENTS.md 不存在 | 从零创建 |
| AGENTS.md 存在 | 读取现有内容，确定变更范围 |

**信息收集**：

| 信息来源 | 收集内容 |
|----------|----------|
| 项目目录结构 | 模块列表、目录组织 |
| pom.xml / package.json | 技术栈和版本 |
| 现有 design.md | 模块职责和接口 |
| 团队信息 | 角色和联系人 |

### Phase 2: 按模板生成/更新

按  模板结构生成或更新 AGENTS.md：

```markdown
# AGENTS.md - [项目名称]

> 最后更新：[日期]
> 更新人：[姓名]
> 版本：v1.0

## 项目概述
[一句话描述项目目标]

## 技术栈
| 类型 | 技术 | 版本 |

## 项目结构
[项目目录结构]

## 核心模块
### [模块名]
- 职责、关键接口、设计文档链接

## 架构约束
### 允许的依赖方向
### 禁止的依赖

## 编码规范
### 命名约定
### 代码风格

## AI 使用约束
### 三级边界
### 上下文来源

## 联系方式
| 角色 | 姓名 | 职责 |

## 变更日志
| 版本 | 日期 | 变更内容 | 变更人 |
```

### Phase 3: 变更类型判定

| 变更类型 | 触发条件 | 审核人 | 生效时间 |
|----------|----------|--------|----------|
| 重大架构变更 | 模块增删、依赖关系变化 | 架构师 | 即时 |
| 技术栈升级 | 框架版本升级 | 架构师 | 即时 |
| 人员变动 | 角色职责调整 | 技术负责人 | 即时 |
| 常规迭代 | 月末复盘会后 | 技术负责人 | 每月 |

### Phase 4: 变更记录

每次更新必须记录变更日志：

```markdown
## 变更日志

| 版本 | 日期 | 变更内容 | 变更人 |
|------|------|----------|--------|
| v1.1 | 2026-04-20 | 新增订单模块描述 | 张三 |
| v1.0 | 2026-04-15 | 初始版本 | 李四 |
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 变更描述 | 文本 | 是 | 需要更新的内容 |
| 变更类型 | 枚举 | 否 | 架构变更/技术栈升级/人员变动/常规迭代 |
| 项目信息 | 文件 | 否 | pom.xml/package.json 等 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| AGENTS.md | Markdown | 更新后的项目地图 |
| 变更摘要 | 文本 | 本次变更的摘要说明 |

## 检查清单

- [ ] 项目概述是否准确？
- [ ] 技术栈版本是否最新？
- [ ] 项目结构是否反映当前目录？
- [ ] 核心模块是否全部列出？
- [ ] 架构约束是否明确？
- [ ] AI 使用约束是否包含三级边界？
- [ ] 变更日志是否记录？

## 与其他 Skills 的关系
