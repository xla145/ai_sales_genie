# AI 生成代码自检清单



## 功能性

- [ ] 代码逻辑是否符合设计文档？
- [ ] 边界条件是否处理完备？（空值、越界、并发）
- [ ] 异常分支是否覆盖？
- [ ] 输入输出是否符合接口契约？
- [ ] 状态流转是否与 design.md 一致？

## 质量性

- [ ] 是否符合团队编码规范？（见 java-code-standards.md）
- [ ] 命名是否清晰一致？（PascalCase/camelCase/UPPER_SNAKE）
- [ ] 是否有不必要的复杂度？（圈复杂度 ≤ 10）
- [ ] 是否有重复代码？（重复率 < 3%）
- [ ] 方法长度是否合理？（≤ 50 行）

## 安全性

- [ ] 是否有 SQL 注入风险？（无字符串拼接 SQL）
- [ ] 是否有 XSS 风险？（用户输入已转义）
- [ ] 敏感数据是否脱敏？（手机号、身份证、密码）
- [ ] 权限校验是否完整？（写操作有权限检查）
- [ ] 密码是否加密存储？（BCrypt/MD5+Salt）

## 性能性

- [ ] 是否有明显的性能瓶颈？（N+1 查询、全表扫描）
- [ ] 数据库查询是否优化？（索引、分页、批量）
- [ ] 是否有内存泄漏风险？（未关闭资源、无限集合）
- [ ] 是否有并发安全问题？（共享状态、竞态条件）

## 测试性

- [ ] 是否提供单元测试？
- [ ] 测试覆盖是否充分？（核心逻辑 ≥ 80%）
- [ ] 边界条件是否有测试？
- [ ] 异常路径是否有测试？
- [ ] 测试命名是否规范？（should{预期}When{条件}）
