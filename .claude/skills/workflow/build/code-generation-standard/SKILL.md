# 代码生成标准约束

## 规范来源

> 本 Skill 基于 Step2 代码生成标准实现

## 触发条件

- AI 生成代码后（自动触发）
- 用户要求"检查代码质量"或"代码自检"
- 代码提交前质量门禁
- MR 创建前的预检

## 工作流

### Phase 1: 必须包含检查

逐项检查生成代码是否包含以下必要元素：

| # | 检查项 | 说明 | 严重级别 |
|---|--------|------|----------|
| 1 | 完整的类/组件定义 | 类结构完整，无缺失方法 | 🚫 P0 |
| 2 | 适当的 JSDoc/JavaDoc 注释 | 类级 + 关键方法注释 | ⚠️ P1 |
| 3 | 类型注解 | TypeScript 严格模式 / Java 泛型 | 🚫 P0 |
| 4 | 异常处理逻辑 | try-catch / 全局异常处理 | 🚫 P0 |
| 5 | 单元测试骨架 | 至少包含核心方法的测试 | ⚠️ P1 |

### Phase 2: 禁止出现检查

逐项检查生成代码是否包含以下禁止元素：

| # | 检查项 | 匹配模式 | 严重级别 |
|---|--------|----------|----------|
| 1 | 硬编码魔法数字 | `if (status == 1)` | 🚫 P0 |
| 2 | 硬编码魔法字符串 | `if (type.equals("ADMIN"))` | 🚫 P0 |
| 3 | TODO 占位符 | `// TODO:` （无验收标准） | ⚠️ P1 |
| 4 | N+1 查询 | 循环内查询数据库 | 🚫 P0 |
| 5 | SQL 拼接 | 字符串拼接 SQL | 🚫 P0 |
| 6 | 未处理的异常 | 空 catch 块 | 🚫 P0 |
| 7 | 无限循环风险 | 无边界 while/for | ⚠️ P1 |

### Phase 3: design.md 一致性校验

如果当前模块存在 design.md，校验代码与设计的一致性：

| # | 校验项 | 说明 | 严重级别 |
|---|--------|------|----------|
| 1 | 接口定义一致 | 代码中的接口签名与 design.md 一致 | 🚫 P0 |
| 2 | 数据结构一致 | 实体字段与 design.md 表设计一致 | 🚫 P0 |
| 3 | 未引入未声明依赖 | 新增依赖在 design.md 中有说明 | ⚠️ P1 |
| 4 | 分层架构一致 | Controller → Service → Repository 分层正确 | 🚫 P0 |
| 5 | 异常处理一致 | 异常场景与 design.md 中定义一致 | ⚠️ P1 |

### Phase 4: 自检清单执行

执行完整自检清单：

```markdown
## AI 生成代码自检报告

**文件**：[文件名]
**检查时间**：[时间]
**检查结果**：[通过 ✅ / 不通过 ❌]

### 功能性
- [ ] 代码逻辑是否符合设计文档？
- [ ] 边界条件是否处理完备？
- [ ] 异常分支是否覆盖？
- [ ] 输入输出是否符合接口契约？

### 质量性
- [ ] 是否符合团队编码规范？
- [ ] 命名是否清晰一致？
- [ ] 是否有不必要的复杂度？
- [ ] 是否有重复代码？

### 安全性
- [ ] 是否有 SQL 注入风险？
- [ ] 是否有 XSS 风险？
- [ ] 敏感数据是否脱敏？
- [ ] 权限校验是否完整？

### 性能性
- [ ] 是否有明显的性能瓶颈？
- [ ] 数据库查询是否优化？
- [ ] 是否有内存泄漏风险？

### 测试性
- [ ] 是否提供单元测试？
- [ ] 测试覆盖是否充分？
- [ ] 边界条件是否有测试？
```

### Phase 5: 结果输出

```markdown
## 代码生成标准检查结果

**检查范围**：[文件列表]
**总体结果**：[✅ 通过 / ❌ 不通过]

### 必须包含 ❌ 缺失
| # | 文件 | 缺失项 | 建议 |
|---|------|--------|------|

### 禁止出现 ❌ 违规
| # | 文件:行号 | 违规项 | 严重级别 | 建议 |
|---|-----------|--------|----------|------|

### design.md 不一致
| # | 文件 | 不一致项 | design.md 定义 | 代码实现 |
|---|------|----------|---------------|----------|

### 自检未通过项
| # | 维度 | 未通过项 | 建议 |
|---|------|----------|------|
```

**不通过处理**：P0 问题必须修复后才能继续，P1 问题记录到技术债务。

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 生成代码 | 文件 | 是 | AI 生成的代码文件 |
| design.md | 文件 | 否 | 当前模块的设计文档 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 检查结果 | ✅/❌ | 是否通过所有检查 |
| 问题列表 | 表格 | 发现的问题及建议 |
| 自检报告 | Markdown | 完整的自检清单 |

## 检查清单

- [ ] 必须包含项是否全部满足？
- [ ] 禁止出现项是否全部通过？
- [ ] design.md 一致性是否通过？
- [ ] 自检清单是否全部通过？
- [ ] P0 问题是否全部修复？
