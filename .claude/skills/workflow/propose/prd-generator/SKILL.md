# PRD 文档生成

## 规范来源

> 本 Skill 基于 PRD 要求实现

## 核心原则

| 原则 | 说明 |
|------|------|
| **简洁实用** | 不要废话、不要战略口号 |
| **具体可执行** | 具体到页面、功能、输入输出、边界条件 |
| **可给开发用** | 能直接给设计和开发使用 |
| **包含验收标准** | 每个功能做成什么样算完成 |

## 触发条件

- 需求确认稿已确认，需要生成 PRD
- 用户要求"生成 PRD"或"写产品需求文档"
- 需求评审前需要标准化文档

## 工作流

### Phase 1: 输入检查

确认输入的完整性：

| 输入项 | 必要性 | 检查内容 |
|--------|--------|----------|
| 需求确认稿 | 必须 | 6 件事是否完整 |
| 项目上下文 | 推荐 | AGENTS.md 中的技术栈和架构约束 |
| 参考竞品 | 可选 | 对标产品的功能清单 |

**需求确认稿不完整时**：
- 标注缺失项
- 建议先完成 `requirement-confirmation-draft`
- 如用户坚持，用 `[待补充]` 占位

### Phase 2: PRD 生成

按 10 要素模板生成 PRD：

```markdown
# [项目名称] 产品需求文档（PRD）

> 版本：v1.0
> 作者：[姓名]
> 日期：[日期]

## 1. 项目背景

[为什么要做，业务痛点，当前现状]

## 2. 项目目标

[解决什么问题、带来什么结果，可量化]

## 3. 目标用户

[是谁、特点是什么、技术背景]

## 4. 核心使用场景

[怎么进入、怎么使用、怎么完成目标]

## 5. 功能范围

### 必做（MVP）
| # | 功能 | 描述 | 验收标准 |
|---|------|------|----------|

### 可选
| # | 功能 | 描述 | 验收标准 |
|---|------|------|----------|

### 暂不做
| # | 功能 | 原因 | 预计迭代 |
|---|------|------|----------|

## 6. 页面结构

[有哪些页面、每个页面做什么]

## 7. 关键流程

[注册、生成、告警处理等核心流程]

## 8. 数据与状态

[核心数据对象、状态流转]

## 9. 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | [如：列表加载 < 2s] |
| 权限 | [如：需要登录，按角色控制] |
| 安全 | [如：敏感数据脱敏] |
| 兼容性 | [如：Chrome 90+, Edge 90+] |
| 可维护性 | [如：遵循团队编码规范] |

## 10. 验收标准

| # | 功能 | 验收标准 | 优先级 |
|---|------|----------|--------|
```

### Phase 3: 质量检查

生成后执行 PRD 质量检查：

- [ ] 每个功能是否有明确的验收标准？
- [ ] 页面结构是否覆盖所有功能？
- [ ] 关键流程是否完整（正常路径 + 异常路径）？
- [ ] 数据对象和状态是否定义？
- [ ] 非功能需求是否具体可测？
- [ ] 是否避免了战略口号和模糊描述？

### Phase 4: 评审建议

输出评审建议：

```markdown
## PRD 评审建议

### ✅ 已覆盖
- [覆盖的要点]

### ⚠️ 建议补充
- [缺失或不够具体的地方]

### 📋 评审前准备
- [ ] 确认功能范围与产品经理对齐
- [ ] 确认技术可行性（需架构师参与）
- [ ] 确认工期评估
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 需求确认稿 | 文本 | 是 | 6 件事确认稿 |
| 项目上下文 | 文本 | 否 | AGENTS.md 内容 |
| 参考竞品 | 文本 | 否 | 对标产品信息 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| PRD 文档 | Markdown | 10 要素产品需求文档 |
| 评审建议 | Markdown | PRD 评审前的准备建议 |
| 缺失项 | 列表 | 需求确认稿中不完整的部分 |

## 检查清单

- [ ] 10 个要素是否全部覆盖？
- [ ] 每个功能是否有验收标准？
- [ ] 验收标准是否具体可测？
- [ ] 是否避免了模糊描述？
- [ ] 功能范围是否与确认稿一致？
- [ ] 非功能需求是否具体？
