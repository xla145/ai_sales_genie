# 标准化任务模板



## 单个任务模板

```markdown
---

### 任务：[任务名称]

| 项目 | 内容 |
|------|------|
| **目标** | [具体要达成的目标] |
| **角色** | [前端 / 后端 / 全栈 / 测试] |
| **优先级** | [P0 必做 / P1 重要 / P2 可选] |
| **预估工时** | [X 人天] |

**输入**：
- [需要的输入，如设计稿、API 文档、对标截图]

**输出**：
- [预期的产出，如可访问的页面、可调用的 API]

**依赖**：
- [前置依赖，如"接口已定义"、"设计风格已确定"]
- 无依赖则标注：无

**完成标准**：
- [ ] [可验收的标准 1，如"PC 和移动端可正常显示"]
- [ ] [可验收的标准 2，如"按钮可点击并跳转"]
- [ ] [可验收的标准 3，如"接口返回正确的数据格式"]

---
```

## 任务清单模板

```markdown
# [项目名称] 开发任务清单


> 创建日期：[日期]
> 总预估：[X] 人天

## 阶段 1：基础框架（MVP 核心）

> 目标：[本阶段要达成的目标]

| 序号 | 任务 | 角色 | 优先级 | 预估 | 依赖 | 完成标准 |
|------|------|------|--------|------|------|----------|
| 1.1 | [任务名] | [角色] | P0 | [X天] | 无 | [标准] |
| 1.2 | [任务名] | [角色] | P0 | [X天] | 1.1 | [标准] |

**阶段验收标准**：
- [ ] [本阶段整体验收条件]

## 阶段 2：功能完善

> 目标：[本阶段要达成的目标]

| 序号 | 任务 | 角色 | 优先级 | 预估 | 依赖 | 完成标准 |
|------|------|------|--------|------|------|----------|

## 阶段 3：可选功能

> 目标：[本阶段要达成的目标]

| 序号 | 任务 | 角色 | 优先级 | 预估 | 依赖 | 完成标准 |
|------|------|------|--------|------|------|----------|

## 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| [风险描述] | [影响范围] | [应对方案] |
```

## 粒度参考

| 任务类型 | 建议粒度 | 示例 |
|----------|----------|------|
| 页面开发 | 单个页面 | "用户列表页面开发" |
| API 开发 | 单个接口 CRUD | "用户管理 API 开发" |
| 组件开发 | 单个可复用组件 | "分页表格组件封装" |
| 数据库 | 单表设计 + 迁移 | "用户表设计与建表" |
| 测试 | 单模块测试 | "用户模块单元测试" |
| 文档 | 单模块文档 | "用户模块 API 文档" |
