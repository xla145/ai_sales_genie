# Vibe Coding 需求访谈

## 规范来源

> 本 Skill 基于 Vibe Coding 需求对齐阶段实现

## 核心原则

> **在进入开发前，先让 AI 真正理解需求，避免"能跑，但不是你要的"**

| 原则 | 说明 |
|------|------|
| **AI 主动提问** | 不是一次性说完，而是让 AI 分轮次追问、持续澄清 |
| **需求访谈模式** | 让 AI 扮演产品经理/解决方案顾问，先问清楚再做 |
| **对齐确认** | AI 复述理解，开发者确认，避免理解偏差 |
| **需求确认稿** | 进入 PRD 和开发前，先输出需求确认稿，对齐 6 件事 |

## 触发条件

- 用户提出一个模糊的需求描述
- 用户说"我想做一个 XX"
- 用户说"帮我梳理需求"
- 用户说"先聊透需求再动手"

**⛔ 重要**：触发本 Skill 后，**绝不直接写代码或输出方案**，必须先完成访谈。

## 工作流

### Phase 1: 开启访谈

收到用户需求后，以资深产品经理身份开启访谈：

```
收到需求：[用户原始描述]

我理解你想做 [复述理解]。在我开始设计方案之前，需要先问清楚几个关键问题，
确保我真正理解你的需求。

让我分几轮来问，每轮只问 2-3 个最关键的问题。
```

### Phase 2: 分轮次追问

#### 第一轮：核心问题（必问）

从以下问题中选择最相关的 2-3 个：

- [ ] 这个功能要解决什么问题？为什么要做？
- [ ] 目标用户是谁？他们的技术背景如何？
- [ ] 核心用户场景是什么？一次完整的使用路径是什么？
- [ ] 第一版最核心的目标是什么？（MVP 思维）

#### 第二轮：功能边界

根据第一轮回答，追问 2-3 个：

- [ ] 需要登录吗？需要权限控制吗？
- [ ] 需要数据库吗？需要持久化什么数据？
- [ ] 是 PC 优先还是移动端优先？
- [ ] 第一版必须做什么？哪些是可选的？哪些明确不做？

#### 第三轮：技术约束

根据前两轮回答，追问 2-3 个：

- [ ] 预期数据量级是多少？
- [ ] 需要实时响应还是异步处理？
- [ ] 需要集成哪些现有系统？
- [ ] 有没有对标的竞品或参考产品？
- [ ] 有没有安全合规要求？

#### 第 N 轮：持续澄清

重复追问直到信息足够：

- 对模糊的回答继续追问
- 对矛盾的信息请求澄清
- 对遗漏的场景补充提问

**追问规则**：
- 每轮不超过 3 个问题
- 优先问最影响产品设计和开发方案的问题
- 用户回答后，简要复述理解，确认无误

### Phase 3: 总结理解

信息足够后，输出理解总结：

```markdown
## 我的理解

### 要解决的问题
[用一句话概括]

### 目标用户
[用户类型和特点]

### 核心场景
[主要使用路径]

### MVP 范围
[第一版必须做的]

### 暂不包含
[明确不做的]

### 不确定的地方
[仍需确认的问题]
```

**等待用户确认**：`以上理解是否正确？有需要补充或修正的地方吗？`

### Phase 4: 输出需求确认稿

用户确认理解后，输出结构化的需求确认稿（6 件事）：

```markdown
# 需求确认稿

## 1. 产品目标
[要解决什么问题、为什么要做、第一版最核心的目标]

## 2. 目标用户
[给谁用、用户类型、技术背景]

## 3. 核心使用场景
[用户怎么进入、主要任务是什么、完整使用路径]

## 4. MVP 范围
[第一版必须做什么]

## 5. 暂不包含
[明确不做的功能、复杂能力]

## 6. 待确认问题
[AI 仍不确定、不同选择会影响方案的地方]
```

**完成标志**：用户确认需求确认稿后，可进入 PRD 生成（`prd-generator`）或任务拆分（`task-decomposition`）。

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 需求描述 | 文本 | 是 | 用户的初始需求描述 |
| 补充信息 | 文本 | 否 | 用户在访谈中提供的额外信息 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 访谈记录 | 对话 | 分轮次的问答记录 |
| 理解总结 | Markdown | AI 对需求的理解总结 |
| 需求确认稿 | Markdown | 结构化的 6 件事确认稿 |

## 检查清单

需求确认稿完成前确认：

- [ ] 产品目标是否清晰？（为什么做、核心目标）
- [ ] 目标用户是否明确？（谁用、技术背景）
- [ ] 核心场景是否完整？（进入→使用→完成）
- [ ] MVP 范围是否界定？（必做 vs 可选 vs 不做）
- [ ] 暂不包含是否明确？（避免范围蔓延）
- [ ] 待确认问题是否标注？（影响方案的不确定项）
