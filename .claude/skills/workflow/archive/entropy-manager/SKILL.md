# 熵管理器

## 规范来源

> 本 Skill 基于 熵管理 (Entropy Management) 实现

## 触发条件

- 周维度：CI/CD 构建后自动触发健康度检查
- 月维度：迭代评审时触发技术债务盘点
- 季度维度：架构评审时触发架构健康度评估
- 用户主动请求代码库健康度报告
- 迭代复盘时需要量化数据支撑

## 工作流

### Phase 1: 代码健康度指标采集

采集 5 项核心健康度指标：

```
健康度指标采集
    │
    ├── 圈复杂度 → 方法内线性独立路径数
    │   ├── 健康阈值：≤10
    │   └── 告警阈值：>15
    │
    ├── 重复代码率 → 重复代码行/总代码行
    │   ├── 健康阈值：<3%
    │   └── 告警阈值：>5%
    │
    ├── 注释率 → 注释行/代码行
    │   ├── 健康阈值：15%-30%
    │   └── 告警阈值：<10% 或 >50%
    │
    ├── 命名一致性 → 命名符合规范的程度
    │   ├── 健康阈值：95%
    │   └── 告警阈值：<90%
    │
    └── 模块内聚度 → 类的职责单一程度
        ├── 健康阈值：>70%
        └── 告警阈值：<50%
```

| 指标 | 定义 | 健康阈值 | 告警阈值 | 采集方式 |
|------|------|----------|----------|----------|
| 圈复杂度 | 方法内线性独立路径数 | ≤10 | >15 | PMD / SonarQube |
| 重复代码率 | 重复代码行/总代码行 | <3% | >5% | CPD / SonarQube |
| 注释率 | 注释行/代码行 | 15%-30% | <10% 或 >50% | 代码统计工具 |
| 命名一致性 | 命名符合规范的程度 | 95% | <90% | Checkstyle |
| 模块内聚度 | 类的职责单一程度 | >70% | <50% | LCOM 分析 |

### Phase 2: 周维度 — CI/CD 结果汇总

```
周维度检查
    │
    ├── CI/CD 构建状态
    │   ├── 构建成功率
    │   ├── 构建失败原因分类
    │   └── 平均构建时间
    │
    ├── 代码规范检查
    │   ├── Checkstyle 违规数量
    │   ├── PMD 告警数量
    │   └── 新增违规趋势
    │
    ├── 测试覆盖率
    │   ├── 核心业务逻辑覆盖率
    │   ├── 工具类/DTO 覆盖率
    │   └── 整体项目覆盖率
    │
    └── 报告推送
        └── 生成周维度健康度摘要
```

### Phase 3: 月维度 — 技术债务盘点

```
月维度检查
    │
    ├── 技术债务登记
    │   ├── 汇总本月新增 P2 问题
    │   ├── 统计技术债务总量
    │   └── 标记超期未处理债务
    │
    ├── 重构优先级排序
    │   ├── 按影响范围排序
    │   ├── 按修复成本排序
    │   └── 按风险等级排序
    │
    └── 纳入迭代计划
        ├── 建议下次迭代修复的债务
        └── 预估修复工时
```

### Phase 4: 季度维度 — 架构健康度评估

```
季度维度检查
    │
    ├── 架构健康度评估
    │   ├── 模块依赖关系图
    │   ├── 循环依赖检测
    │   └── 分层违规统计
    │
    ├── 模块边界审查
    │   ├── 职责越界检测
    │   ├── 接口稳定性评估
    │   └── API 兼容性检查
    │
    └── 依赖关系梳理
        ├── 外部依赖版本检查
        ├── 废弃依赖清理
        └── 安全漏洞依赖扫描
```

### Phase 5: 输出报告

```markdown
## 📊 熵管理报告

**报告周期**：[周/月/季度]
**报告日期**：YYYY-MM-DD
**项目**：[项目名称]

### 1. 健康度指标总览

| 指标 | 当前值 | 健康阈值 | 告警阈值 | 状态 | 趋势 |
|------|--------|----------|----------|------|------|
| 圈复杂度 | 8.2 | ≤10 | >15 | ✅ 健康 | → 稳定 |
| 重复代码率 | 2.1% | <3% | >5% | ✅ 健康 | ↓ 改善 |
| 注释率 | 22% | 15%-30% | <10%/>50% | ✅ 健康 | → 稳定 |
| 命名一致性 | 93% | 95% | <90% | ⚠️ 告警 | ↓ 退化 |
| 模块内聚度 | 75% | >70% | <50% | ✅ 健康 | → 稳定 |

### 2. CI/CD 结果（周维度）

| 指标 | 本周 | 上周 | 变化 |
|------|------|------|------|
| 构建成功率 | 95% | 98% | ↓ -3% |
| Checkstyle 违规 | 12 | 8 | ↑ +4 |
| 测试覆盖率 | 78% | 76% | ↑ +2% |

### 3. 技术债务登记（月维度）

| ID | 描述 | 位置 | 复杂度 | 建议迭代 | 状态 |
|----|------|------|--------|----------|------|
| TD-001 | UserService 方法过长 | service/UserService.java | 中 | v2.1 | 待处理 |

### 4. 架构评估（季度维度）

| 维度 | 评分 | 说明 |
|------|------|------|
| 模块依赖 | A | 无循环依赖，分层清晰 |
| 接口稳定性 | B | 2 个接口本月有破坏性变更 |
| 依赖安全 | C | 3 个依赖有已知安全漏洞 |

### 5. 改进建议

1. [建议 1]
2. [建议 2]
3. [建议 3]
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 报告维度 | 枚举 | 否 | weekly（周）/ monthly（月）/ quarterly（季度），默认 weekly |
| 模块范围 | 文本 | 否 | 指定检查的模块/包，默认全项目 |
| 基线日期 | 日期 | 否 | 对比基线日期，用于趋势分析 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 健康度指标 | 表格 | 5 项指标当前值和趋势 |
| CI/CD 摘要 | 表格 | 构建状态和规范检查结果 |
| 技术债务登记 | 表格 | 债务清单和优先级 |
| 架构评估 | 表格 | 架构健康度评分 |
| 改进建议 | 列表 | 可操作的改进建议 |

## 检查清单

- [ ] 圈复杂度是否超过健康阈值？
- [ ] 重复代码率是否超过健康阈值？
- [ ] 注释率是否在合理范围？
- [ ] 命名一致性是否达标？
- [ ] 模块内聚度是否达标？
- [ ] CI/CD 构建成功率是否正常？
- [ ] 测试覆盖率是否达标？
- [ ] 技术债务是否有超期未处理？
- [ ] 是否有循环依赖？
- [ ] 是否有安全漏洞依赖？
