# 健康度指标采集清单

## 1. 圈复杂度

| 采集方式 | 工具 | 配置 |
|----------|------|------|
| PMD | `mvn pmd:check` | 规则集：`category/java/design.xml/CyclomaticComplexity` |
| SonarQube | 内置分析 | 阈值：≤10（方法级） |

### 检查要点

- [ ] 单个方法圈复杂度是否 ≤10？
- [ ] 是否有方法圈复杂度 >15（告警）？
- [ ] 高复杂度方法是否有重构计划？

---

## 2. 重复代码率

| 采集方式 | 工具 | 配置 |
|----------|------|------|
| CPD (PMD) | `mvn pmd:cpd-check` | 最小重复行数：50 |
| SonarQube | 内置分析 | 阈值：<3% |

### 检查要点

- [ ] 项目整体重复代码率是否 <3%？
- [ ] 是否有 >5% 的重复代码（告警）？
- [ ] 新增代码是否引入了重复？

---

## 3. 注释率

| 采集方式 | 工具 | 配置 |
|----------|------|------|
| 代码统计 | `cloc` 或 SonarQube | 范围：15%-30% |

### 检查要点

- [ ] 注释率是否在 15%-30% 范围内？
- [ ] 是否有 <10% 的注释率（告警）？
- [ ] 是否有 >50% 的注释率（过度注释）？
- [ ] 公共 API 是否有 JavaDoc？

---

## 4. 命名一致性

| 采集方式 | 工具 | 配置 |
|----------|------|------|
| Checkstyle | `mvn checkstyle:check` | 规则：Google Java Style |
| SonarQube | 内置分析 | 阈值：95% |

### 检查要点

- [ ] 类名是否使用 PascalCase？
- [ ] 方法名是否使用 camelCase？
- [ ] 常量是否使用 UPPER_SNAKE_CASE？
- [ ] 包名是否全小写？
- [ ] 命名一致性是否 ≥95%？

---

## 5. 模块内聚度

| 采集方式 | 工具 | 配置 |
|----------|------|------|
| LCOM 分析 | SonarQube / JDepend | 阈值：>70% |

### 检查要点

- [ ] 类的职责是否单一？
- [ ] LCOM 值是否 >70%（高内聚）？
- [ ] 是否有 LCOM <50% 的类（低内聚，告警）？
- [ ] 上帝类是否需要拆分？
