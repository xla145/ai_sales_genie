# 反馈循环收集器

## 规范来源

> 本 Skill 基于 反馈循环机制实现

## 触发条件

- 迭代结束时进行迭代复盘
- 月度 AI 使用复盘
- 季度架构评审
- 用户主动请求反馈汇总
- CI/CD 流水线持续反馈

## 工作流

### Phase 1: 即时反馈收集

收集开发过程中的即时反馈数据：

```
即时反馈来源
    │
    ├── CI/CD 流水线
    │   ├── 构建状态（成功/失败/不稳定）
    │   ├── 测试结果（通过率、失败用例）
    │   ├── 代码规范检查（违规数量、类型分布）
    │   └── 覆盖率变化（新增/下降）
    │
    ├── AI 自检
    │   ├── 代码质量评估（code-generation-standard 检查结果）
    │   ├── 安全扫描（data-security-guard 扫描结果）
    │   ├── 架构约束（architecture-constraint-checker 检查结果）
    │   └── 即时验收（vibe-verify 验收结果）
    │
    └── 代码审查
        ├── 审查意见数量和分布
        ├── P0/P1/P2 问题统计
        └── 修复率和修复时长
```

| 反馈来源 | 反馈内容 | 处理方式 |
|----------|----------|----------|
| CI/CD 流水线 | 构建状态、测试结果、代码规范 | 自动通知开发者 |
| AI 自检 | 代码质量评估、安全扫描 | 开发者处理 |
| 代码审查 | 审查意见、改进建议 | 开发者修订 |

### Phase 2: 迭代反馈收集

收集迭代维度的反馈数据：

```
迭代反馈来源
    │
    ├── AI 使用效果评估
    │   ├── AI 生成代码的采纳率
    │   ├── AI 建议的准确率
    │   ├── AI 辅助节省的工时估算
    │   └── AI 使用的痛点统计
    │
    ├── 规范优化反馈
    │   ├── 规范执行中的困难点
    │   ├── 规范覆盖的盲区
    │   ├── 规范冲突的案例
    │   └── 规范更新的需求
    │
    └── 技能迭代反馈
        ├── 技能使用频率统计
        ├── 技能效果评估
        ├── 技能缺失需求
        └── 技能优化建议
```

| 反馈类型 | 内容 | 输出物 |
|----------|------|--------|
| 迭代复盘 | 本迭代 AI 使用效果评估 | 迭代复盘报告 |
| 规范优化 | 基于反馈的规范更新 | AGENTS.md 修订 |
| 技能迭代 | Claude Skills 更新 | Skill 版本升级 |

### Phase 3: 数据汇总与分析

将收集到的反馈数据进行汇总分析：

```
数据汇总
    │
    ├── 定量指标
    │   ├── CI/CD 构建成功率趋势
    │   ├── 测试覆盖率变化趋势
    │   ├── P0/P1 问题数量趋势
    │   ├── AI 代码采纳率趋势
    │   └── 技术债务增长率
    │
    ├── 定性反馈
    │   ├── 开发者满意度
    │   ├── AI 使用体验评价
    │   ├── 规范执行困难点
    │   └── 改进建议汇总
    │
    └── 对比分析
        ├── 与上一迭代对比
        ├── 与基线目标对比
        └── 行业基准对比（可选）
```

### Phase 5: 输出 Skills 更新建议

基于反馈数据，生成 Claude Skills 更新建议：

| 建议类型 | 说明 | 优先级 |
|----------|------|--------|
| 新增 Skill | 缺失的能力需求 | 按使用频率排序 |
| 更新 Skill | 现有 Skill 的优化 | 按效果评分排序 |
| 废弃 Skill | 使用率极低的 Skill | 按使用次数排序 |
| 规则更新 | 全局规则的调整 | 按执行困难度排序 |

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 迭代信息 | 文本 | 是 | Sprint 编号和周期 |
| 反馈数据 | 文件 | 否 | CI/CD 日志、审查记录、AI 使用记录 |
| 报告类型 | 枚举 | 否 | sprint（迭代）/ monthly（月度）/ quarterly（季度），默认 sprint |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 即时反馈汇总 | 表格 | CI/CD、AI 自检、代码审查数据 |
| AI 使用效果 | 表格 | 采纳率、准确率、节省工时 |
| 规范执行情况 | 表格 | 执行率和困难点 |
| Skills 使用统计 | 表格 | 使用频率和效果评分 |
| 改进建议 | 列表 | Skills 更新 + 规范优化 + 下迭代重点 |
