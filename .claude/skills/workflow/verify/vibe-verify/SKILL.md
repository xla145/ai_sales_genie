# Vibe Coding 即时验收

## 规范来源

> 本 Skill 基于 Step3 即时验收实现
> 氛围编程核心理念："边开发边打磨，每个功能都要验收"

## 核心流程

```
AI 生成代码 → 开发者即时验收
                  │
                  ├── 符合预期 → 进入下一任务 ✅
                  └── 不符合预期 → AI 重新做 🔄
```

## 触发条件

- AI 完成一个功能/模块的代码生成后
- 用户要求"验收"或"检查这个功能"
- 每个开发任务完成后（即时验收原则）
- 代码修复后重新验证

## 工作流

### Phase 1: 功能性验收

| # | 检查点 | 验收方式 | 通过标准 |
|---|--------|----------|----------|
| 1 | 页面/功能是否符合需求 | 实际运行操作 | 与 PRD/任务描述一致 |
| 2 | 核心流程是否走通 | 完整执行主流程 | 无阻断性错误 |
| 3 | 数据是否正确 | 输入→输出验证 | 返回数据与预期一致 |
| 4 | 接口是否可调通 | API 测试 | 返回正确状态码和数据 |

### Phase 2: UI/交互验收

| # | 检查点 | 验收方式 | 通过标准 |
|---|--------|----------|----------|
| 1 | 样式是否符合设计 | 对照设计稿/参考 | 布局、颜色、字体一致 |
| 2 | 交互是否符合预期 | 点击、输入操作 | 按钮响应、表单提交正常 |
| 3 | 加载状态是否展示 | 触发异步操作 | Loading 动画/骨架屏 |
| 4 | 空状态是否有提示 | 清空数据查看 | 空状态占位图/文案 |

### Phase 3: 用户体验验收

| # | 检查点 | 验收方式 | 通过标准 |
|---|--------|----------|----------|
| 1 | 操作流程是否顺畅 | 完整走一遍流程 | 无多余步骤、无卡顿 |
| 2 | 错误提示是否友好 | 输入错误数据 | 提示清晰、可操作 |
| 3 | 成功反馈是否明确 | 完成操作 | 有成功提示/跳转 |
| 4 | 返回/取消是否安全 | 中途退出 | 有确认提示、数据不丢失 |

### Phase 4: 边界处理验收

| # | 检查点 | 验收方式 | 通过标准 |
|---|--------|----------|----------|
| 1 | 必填项为空 | 不填必填字段提交 | 有校验提示 |
| 2 | 超长输入 | 输入超长文本 | 有长度限制/截断 |
| 3 | 特殊字符 | 输入 `<script>` 等 | 不执行、正确显示 |
| 4 | 并发操作 | 快速重复点击 | 无重复提交 |
| 5 | 网络异常 | 模拟断网 | 有错误提示、可重试 |

### Phase 5: 响应性验收

| # | 检查点 | 验收方式 | 通过标准 |
|---|--------|----------|----------|
| 1 | PC 端是否正常 | Chrome/Edge 浏览器 | 布局不错乱 |
| 2 | 移动端是否正常 | 手机/模拟器 | 响应式适配 |
| 3 | 大数据量是否正常 | 加载大量数据 | 不卡顿、有分页 |

### Phase 6: 验收结论

根据验收结果输出结论：

```markdown
## 即时验收报告

**验收任务**：[任务名称]
**验收时间**：[时间]
**验收结果**：[✅ 通过 / ❌ 不通过]

### 通过项 ✅
| 维度 | 通过项 |
|------|--------|
| 功能性 | [列表] |
| UI/交互 | [列表] |
| 用户体验 | [列表] |
| 边界处理 | [列表] |
| 响应性 | [列表] |

### 未通过项 ❌
| 维度 | 问题描述 | 严重级别 | 处理方式 |
|------|----------|----------|----------|
| [维度] | [问题] | P0/P1/P2 | [重新做/修复/记录] |

### 处理决策
- P0 问题：**立即重新做**，不进入下一任务
- P1 问题：**当次修复**，修复后重新验收
- P2 问题：**记录到技术债务**，可进入下一任务
```

## 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 生成代码 | 文件 | 是 | AI 生成的代码 |
| 验收标准 | 文本 | 是 | 任务的完成标准 |
| PRD/设计参考 | 文本 | 否 | 用于对照验收 |

## 输出

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 验收结果 | ✅/❌ | 是否通过验收 |
| 验收报告 | Markdown | 5 维度验收详情 |
| 问题列表 | 表格 | 未通过项及处理方式 |
