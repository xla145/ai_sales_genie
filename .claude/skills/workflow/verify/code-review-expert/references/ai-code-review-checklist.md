# AI 生成代码专项审查清单

## 1. 代码验证

| 检查项 | 说明 | 判定标准 |
|--------|------|----------|
| API 存在性 | AI 引用的 API/方法是否真实存在 | 检查官方文档和源码 |
| 库/依赖存在性 | AI 引入的依赖是否真实可用 | 检查 Maven Central / npm |
| 配置正确性 | AI 生成的配置是否正确 | 对照官方文档验证 |
| 逻辑合理性 | AI 生成的业务逻辑是否合理 | 人工审查业务正确性 |

## 2. 人工审核签字

| 检查项 | 说明 | 要求 |
|--------|------|------|
| 审核确认 | AI 生成代码是否经过人工审核 | 必须有人工审核记录 |
| 测试验证 | AI 生成代码是否通过测试 | 必须有测试覆盖 |
| 规范符合 | AI 生成代码是否符合团队规范 | 对照编码规范检查 |

## 3. 幻觉检测

| 幻觉类型 | 表现 | 检测方法 |
|----------|------|----------|
| API 幻觉 | 调用不存在的方法 | IDE 报错 / 编译失败 |
| 库幻觉 | 引入不存在的依赖 | 构建失败 |
| 配置幻觉 | 使用不存在的配置项 | 运行时报错 |
| 逻辑幻觉 | 生成看似合理但错误的逻辑 | 人工审查 + 测试 |

## 4. 规范符合度

| 检查项 | 说明 | 规范来源 |
|--------|------|----------|
| 编码规范 | 是否符合 Java 编码规范 | java-code-standards.md |
| 安全规范 | 是否符合安全编码规范 | security.md |
| 测试规范 | 是否符合测试规范 | testing.md |
| 架构规范 | 是否符合架构约束 | AGENTS.md / design.md |

## 5. AI 代码标识

| 要求 | 说明 |
|------|------|
| 标注 AI 生成 | 在 commit message 或代码注释中标注 AI 生成 |
| 人工审核记录 | 在 MR 描述中记录人工审核结果 |
| 测试覆盖 | AI 生成代码必须有对应的测试用例 |
