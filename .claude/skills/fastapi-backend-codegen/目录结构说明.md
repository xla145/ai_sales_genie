## 各目录说明

### 1. common/ - 公共模块

**用途**：包含可直接导入使用的公共模块代码

**文件说明**：

**核心模块：**
- `db.py` - 数据库连接和事务管理
  - `async_session` - 异步数据库会话
  - `async_transaction` - 事务装饰器
  - `content_transaction` - 事务上下文管理器

- `redis.py` - Redis连接和操作
  - `redis_pool` - Redis连接池实例
  - `MyAsyncRedis` - 异步Redis客户端（自动JSON序列化）
  - `RedisPool` - Redis连接池管理类

- `config.py` - 应用配置
  - `Settings` - 配置类
  - `settings` - 全局配置实例

- `deps.py` - API依赖注入
  - `get_db()` - 数据库会话依赖
  - `get_redis()` - Redis连接依赖

**异常和工具：**
- `exceptions.py` - 异常类定义
  - `BusinessException` - 业务异常
  - `IdNotExist` - ID不存在异常
  - `ValidationException` - 验证异常
  - 等其他异常类

- `corelibs.py` - 核心库说明文档
  - `custom_router` - 自定义路由
  - `logger` - 日志模块
  - `codes` - 状态码定义
  - `local` - 本地上下文
  - `consts` - 常量定义

- `utils.py` - 工具函数说明文档
  - `response` - 响应工具
  - `context` - 上下文管理
  - `serialize` - 序列化工具
  - `common` - 通用工具函数

**Schema和应用：**
- `schemas.py` - Schema基础类
  - `BaseSchema` - 所有Schema的基础类
  - `Response` - 统一响应格式
  - `PageResponse` - 分页响应格式

- `main.py` - 应用入口文件模板
  - `create_app()` - 创建FastAPI应用
  - `lifespan` - 应用生命周期管理

- `base_model_usage.md` - Base模型使用指南
  - 公共字段说明
  - CRUD方法详细说明（get, create, update, delete等）
  - 自定义查询方法示例
  - 重要注意事项
  - 完整使用示例

**初始化模块：**
- `init_modules.py` - 初始化模块说明文档
  - `routers` - 路由初始化
  - `exception` - 异常处理初始化
  - `cors` - CORS初始化
  - `middleware` - 中间件初始化

**文档：**
- `usage_examples.md` - 详细使用示例
- `imports_reference.md` - 导入参考文档（所有模块的导入方式）

**使用方式**：
```python
# 数据库连接
from app.db.sqlalchemy import async_session, async_transaction

# Redis连接
from app.db.redis import redis_pool

# 配置
from app.config import settings

# API依赖
from app.api.deps import get_db, get_redis

# 异常处理
from app.exceptions.exceptions import BusinessException, IdNotExist

# 路由
from app.corelibs.custom_router import APIRouter

# 响应
from app.utils.response import HttpResponse

# Schema基础类
from app.schemas.common import BaseSchema

# Base模型（所有Model必须继承）
from app.models.base import Base

# Base模型提供了完整的CRUD方法：
# - get(id): 根据ID查询
# - create(params): 创建记录
# - update(params): 更新记录
# - delete(id): 删除记录（默认软删除）
# - pagination(stmt): 分页查询
# 等等...

# 详细使用说明请查看 common/base_model_usage.md

# 日志
from app.corelibs.logger import logger

# 状态码
from app.corelibs.codes import CodeEnum
```

详细导入方式请查看 `common/imports_reference.md`

## 代码生成流程

### 步骤1：使用模板生成代码

1. 从 `templates/` 复制模板文件
2. 替换占位符
3. 根据业务需求添加字段和方法

### 步骤2：导入公共模块

在生成的代码中导入公共模块：

```python
# Model层
from app.models.base import Base
from app.db.sqlalchemy import async_transaction  # Base已包含，无需手动导入

# Service层
from app.models.user import User as UserModel
from app.db.redis import redis_pool  # 如需使用缓存
from app.config import settings  # 如需使用配置

# API层
from app.corelibs.custom_router import APIRouter
from app.utils.response import HttpResponse
```

### 步骤3：参考示例

查看 `examples/complete_crud_example.md` 了解完整实现。

### 步骤4：检查清单

使用 `SKILL.md` 中的检查清单验证代码。

## 文件映射关系

生成代码时，模板文件对应到实际项目位置：

| 模板文件 | 实际位置 | 说明 |
|---------|---------|------|
| `templates/model_template.py` | `app/models/{model_name}.py` | Model层代码 |
| `templates/service_template.py` | `app/services/{model_name}_service.py` | Service层代码 |
| `templates/schema_template.py` | `app/schemas/{model_name}.py` | Schema层代码 |
| `templates/api_template.py` | `app/api/v1/{model_name}s.py` | API层代码 |
| `common/db.py` | `app/db/sqlalchemy.py` | 数据库连接（已存在） |
| `common/redis.py` | `app/db/redis.py` | Redis连接（已存在） |
| `common/config.py` | `app/config.py` | 配置（已存在） |
| `common/deps.py` | `app/api/deps.py` | API依赖注入（已存在） |
| `common/exceptions.py` | `app/exceptions/exceptions.py` | 异常类（已存在） |
| `common/corelibs.py` | `app/corelibs/` | 核心库说明（已存在） |
| `common/utils.py` | `app/utils/` | 工具函数说明（已存在） |
| `common/schemas.py` | `app/schemas/common.py` | Schema基础类（已存在） |
| `common/main.py` | `backend/main.py` | 应用入口（已存在） |
| `common/base_model.py` | `app/models/base.py` | Base模型说明（已存在） |

## 注意事项

1. **公共模块已存在**：`common/` 目录下的模块在实际项目中已经存在，生成代码时直接导入使用即可，不需要复制代码。

2. **模板占位符**：替换模板中的占位符时注意大小写：
   - `{ModelName}` - 首字母大写的模型名（如 User）
   - `{model_name}` - 小写的模型名（如 user）
   - `{table_name}` - 表名（如 users）
   - `{api_prefix}` - API路径前缀（如 users）

3. **目录结构**：确保生成的文件放在正确的目录下，遵循项目的目录结构规范。

4. **导入路径**：生成的代码中使用相对导入或绝对导入，确保导入路径正确。
