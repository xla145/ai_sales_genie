MySQL 管元数据和过程记录，文件系统/对象存储管代码产物。

不要把所有源码都塞 MySQL。MySQL 存索引、状态、日志、对话、patch、artifact 元信息；真实文件放本地磁盘、S3、OSS、MinIO。

最终架构

FastAPI Runner
  ├─ Hermes Agent Client
  ├─ MySQL
  │   ├─ projects
  │   ├─ project_sessions
  │   ├─ project_runs
  │   ├─ requirements
  │   ├─ run_messages
  │   ├─ run_events
  │   ├─ tool_calls
  │   ├─ file_changes
  │   └─ artifacts
  │
  └─ Storage
      ├─ /projects/{project_id}/current
      ├─ /projects/{project_id}/runs/{run_id}
      └─ artifact.zip

表设计核心

1. projects：项目表

CREATE TABLE projects (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id VARCHAR(64) NOT NULL UNIQUE,
  user_id VARCHAR(64),
  name VARCHAR(255),
  description TEXT,
  status VARCHAR(32) NOT NULL DEFAULT 'active',
  current_version INT NOT NULL DEFAULT 0,
  storage_path VARCHAR(1024),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);


⸻

2. project_sessions：项目 Session

CREATE TABLE project_sessions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  session_id VARCHAR(64) NOT NULL UNIQUE,
  project_id VARCHAR(64) NOT NULL,
  hermes_session_id VARCHAR(128),
  memory_md MEDIUMTEXT,
  project_index JSON,
  message_summary MEDIUMTEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_project_id (project_id)
);

这里存：

memory.md
project_index.json
压缩后的长期对话摘要

不建议存无限长完整上下文作为每次 prompt。

⸻

3. project_runs：每次执行

CREATE TABLE project_runs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  run_id VARCHAR(64) NOT NULL UNIQUE,
  project_id VARCHAR(64) NOT NULL,
  session_id VARCHAR(64) NOT NULL,
  run_type VARCHAR(32) NOT NULL,
  user_message MEDIUMTEXT,
  status VARCHAR(32) NOT NULL DEFAULT 'queued',
  current_step VARCHAR(255),
  progress INT DEFAULT 0,
  error_message MEDIUMTEXT,
  input_snapshot_path VARCHAR(1024),
  output_snapshot_path VARCHAR(1024),
  patch_path VARCHAR(1024),
  started_at DATETIME NULL,
  finished_at DATETIME NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_project_id (project_id),
  INDEX idx_session_id (session_id),
  INDEX idx_status (status)
);

run_type：

create
modify
fix
test
deploy

4. requirements：需求拆解表

CREATE TABLE requirements (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  requirement_id VARCHAR(64) NOT NULL UNIQUE,
  project_id VARCHAR(64) NOT NULL,
  run_id VARCHAR(64),
  parent_id VARCHAR(64),
  title VARCHAR(255) NOT NULL,
  description TEXT,
  requirement_type VARCHAR(32),
  priority VARCHAR(32),
  status VARCHAR(32) DEFAULT 'pending',
  acceptance_criteria JSON,
  estimated_complexity VARCHAR(32),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_project_id (project_id),
  INDEX idx_run_id (run_id),
  INDEX idx_parent_id (parent_id)

用于保存：

用户需求
拆解后的任务
验收标准
实现状态

5. run_messages：执行对话

CREATE TABLE run_messages (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  message_id VARCHAR(64) NOT NULL UNIQUE,
  project_id VARCHAR(64) NOT NULL,
  run_id VARCHAR(64) NOT NULL,

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_run_id (run_id),
  INDEX idx_project_id (project_id)
);

role：

system
user
assistant
tool
planner
runner

这里保存每次给 Hermes 的关键输入输出。

6. run_events：SSE 日志事件

CREATE TABLE run_events (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  event_id VARCHAR(64) NOT NULL UNIQUE,
  project_id VARCHAR(64) NOT NULL,
  run_id VARCHAR(64) NOT NULL,
  event_type VARCHAR(64) NOT NULL,
  level VARCHAR(16) DEFAULT 'info',
  message TEXT,
  payload JSON,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_run_id_id (run_id, id),
  INDEX idx_project_id (project_id)
);

例如：

{
  "event_type": "patch_created",
  "message": "patch 已生成",
  "payload": {
    "path": "runs/run_001/patch.diff"
  }
}

SSE 可以从这张表按 id 增量读。

执行流程入库

1. 创建 project
   -> projects
   -> project_sessions

2. 用户提交需求
   -> project_runs
   -> run_messages

3. 需求拆解
   -> requirements
   -> run_events

4. Hermes 规划相关文件
   -> run_messages
   -> tool_calls
   -> run_events

5. Hermes 生成 patch
   -> run_messages
   -> artifacts
   -> run_events

6. Runner apply patch
   -> tool_calls
   -> file_changes

7. build/test
   -> tool_calls
   -> run_events

8. 保存最终结果
   -> artifacts
   -> projects.current_version + 1
   -> project_sessions.memory_md / project_index

9. run 完成
   -> project_runs.status = success

关键规则

MySQL 存状态、索引、日志、对话、patch 元信息
Storage 存源码、zip、manifest、snapshot
每个 run 保存输入、输出、patch、日志
每个 project 保存 current 最新版本
每次 run 完成后更新 session memory 和 project_index

最终一句话

你这个系统可以定义成：

MySQL = 项目管理 + 执行过程 + 可追溯日志
Storage = 实际代码和产物
Hermes = 单次智能规划和代码修改
Runner = 调度、隔离、索引、patch、入库、归档
