## 2. 页面目标与使用场景

### 2.1 页面目标

车辆管理模块是智能物流系统的核心资源管理模块，主要目标包括：

1. **车辆信息管理**：统一管理所有运输车辆的基础信息，包括车牌、车型、载重、GPS设备等
2. **车辆状态监控**：实时监控车辆状态（空闲、运输中、维修中、离线），为调度提供决策依据
3. **车辆任务追踪**：关联查询车辆当前运输任务和历史任务，支持调度优化
4. **车辆增删改**：支持新增、编辑、删除车辆信息，保持数据准确性

## 7. 业务状态与状态流转

### 7.1 车辆状态流转

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |
|----------|----------|----------|----------|------------|----------|----------|
| 空闲 | idle | 车辆可用 | 新增车辆、任务完成、维修完成 | 派单、运输中、维修 | 运输中、维护中 | - |
| 运输中 | transporting | 执行运输任务 | 调度派单确认 | 更新状态、查看轨迹 | 空闲 | 异常时进入异常流程 |
| 维修中 | maintaining | 车辆维保中 | 人工标记维修 | 查看详情 | 空闲 | - |
| 离线 | offline | 设备离线或停用 | GPS离线、人工标记停用 | 启用、查看详情 | 空闲 | 检查设备 |

### 7.2 状态变更规则

1. **自动变更**：系统根据运输任务状态自动更新车辆状态
2. **手动变更**：调度员可手动将车辆状态设置为维修中或离线
3. **变更记录**：状态变更需记录变更时间、变更人、变更原因

## 9. 交互逻辑

### 9.1 车辆列表页初始化

- **默认展示**：加载第一页数据，每页20条
- **默认筛选**：状态默认为全部（包含所有状态）
- **默认排序**：按创建时间倒序排列（最新在前）
- **初始化异常**：加载失败显示错误提示和重试按钮

### 9.2 查询与筛选

- **筛选项**：
  - 车牌号：模糊匹配输入
  - 车型：下拉选择（全部、小型货车、中型货车、大型货车、重型货车）
  - 状态：下拉选择（全部、空闲、运输中、维修中、离线）
  - 所属公司：下拉选择（全部、公司列表）
- **查询规则**：点击查询按钮后组合所有筛选项进行查询
- **重置规则**：点击重置按钮清空所有筛选项，重新查询全部数据
- **无结果状态**：显示"暂无数据"空状态插图，提供"重置筛选"按钮

### 9.3 新增车辆

- **表单字段**：车牌号（必填）、车型（必填）、载重（必填）、车辆颜色、所属公司（必填）、GPS设备号、SIM卡号
- **校验规则**：
  - 车牌号：必填，格式校验（如：京A12345），唯一性校验
  - 车型：必填，下拉选择
  - 载重：必填，大于0的数字
  - 所属公司：必填，下拉选择
- **提交流程**：填写表单 → 校验 → 提交 → 保存成功 → 跳转列表页
- **重复提交处理**：提交按钮防重复点击，提交中状态禁用按钮

### 9.4 编辑车辆

- **数据回填**：页面加载时自动填充现有数据
- **可编辑字段**：除车牌号外均可编辑
- **校验规则**：同新增车辆（车牌号除外）
- **提交流程**：修改表单 → 校验 → 提交 → 保存成功 → 跳转详情页

### 9.5 删除车辆

- **前置判断**：只有空闲或离线状态的车辆可删除
- **删除确认**：弹出确认框，提示"确定要删除该车辆吗？删除后无法恢复。"
- **删除结果**：删除成功刷新列表，删除失败提示原因

### 9.6 详情与跳转

- **列表→详情**：点击表格中的车牌号或查看按钮，跳转详情页
- **详情→编辑**：点击编辑按钮，跳转编辑页
- **返回规则**：点击返回或浏览器返回，返回列表页并保留筛选条件
- **面包屑**：首页 > 车辆管理 > 车辆详情

## 15. 原型生成说明

### 15.1 页面布局建议

- **列表页布局**：采用标准管理后台布局，左侧菜单+顶部导航+内容区
- **列表结构**：筛选区（可收起展开）+表格区（支持横向滚动）+分页区
- **详情页布局**：面包屑导航+详情内容区+操作按钮区
- **表单页布局**：标题+表单区+底部按钮区

### 15.2 模拟数据要求

提供15+辆车辆模拟数据，包含以下字段：

| 字段 | 示例数据 |
|------|----------|
| 车牌号 | 京A12345、京B67890、京C11111、沪D22222等 |
| 车型 | 小型货车、中型货车、大型货车、重型货车 |
| 载重 | 1.5吨、5吨、12吨、25吨等 |
| 状态 | 空闲（8辆）、运输中（4辆）、维修中（2辆）、离线（1辆） |
| 所属公司 | 物流公司A、货运公司B、运输公司C等 |
| GPS设备 | GPS001、GPS002等，部分显示"未绑定" |

### 15.3 关键交互优先级

1. **P0-必须**：车辆列表筛选、分页、新增车辆表单提交、编辑车辆表单提交
2. **P1-重要**：车辆状态切换、删除确认、详情查看
3. **P2-一般**：导出功能、打印功能

### 15.4 需要重点表达的业务逻辑

1. **车辆状态可视化**：不同状态使用不同颜色标签，直观展示车辆当前状态
2. **状态与操作关联**：运输中车辆禁止编辑和删除，需有明显提示
3. **任务关联展示**：详情页展示当前运输任务，支持快速跳转

### 15.5 需要避免的原型误差

1. 禁止使用"功能开发中"等空白占位
2. 禁止表单只展示无交互
3. 禁止状态筛选无效果
4. 禁止新增/编辑后无反馈提示
5. 禁止删除无确认弹窗
6. 禁止详情页无操作入口

### 15.6 页面跳转关系

```
车辆列表
├── 新增车辆 → 新增车辆页（表单） → 保存成功 → 返回列表
├── 查看 → 车辆详情页 → 返回列表
├── 编辑 → 编辑车辆页（表单） → 保存成功 → 返回详情/列表
└── 删除 → 确认弹窗 → 确认 → 刷新列表

车辆详情
├── 编辑 → 编辑车辆页
├── 返回 → 返回列表
└── 查看任务 → 任务详情弹窗
```
