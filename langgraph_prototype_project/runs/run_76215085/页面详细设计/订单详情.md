## 2. 页面目标与使用场景

### 2.1 页面目标

展示订单全量信息，包括发货信息、承运信息、运输轨迹、签收状态等，支持订单的编辑、取消、派单、复制等核心操作。

### 2.2 使用场景

1. **客服人员**：查看订单详情，为客户解答物流问题；编辑未完成订单信息；取消无法执行的订单；复制订单快速新建
2. **调度员**：查看订单详情，进行智能派单或手动派单；调整已派单任务；监控运输状态
3. **仓库管理员**：查看订单对应的货物信息，进行装车确认、签收确认
4. **管理人员**：查看所有订单详情，审核异常订单，监控整体运营情况

### 2.3 触发入口

- 订单列表页点击某条订单的"查看"按钮
- 首页待办事项中的订单点击
- 智能调度页面的订单详情入口
- 运输轨迹页面的关联订单点击

### 2.4 输出结果

- 完整展示订单的所有信息
- 支持订单状态下的可用操作
- 记录操作日志并提供反馈

## 7. 业务状态与状态流转

### 7.1 订单状态

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |
| --- | --- | --- | --- | --- | --- | --- |
| 待派单 | PENDING | 订单已创建，等待调度派单 | 新建订单完成 | 智能派单、手动派单、编辑、取消 | 已派单、运输中、已取消 | 派单失败保持原状态 |
| 已派单 | DISPATCHED | 订单已分配车辆和司机 | 派单完成 | 查看详情、查看轨迹、取消（需调度确认） | 运输中、已取消 | 车辆故障可重新派单 |
| 运输中 | IN_TRANSIT | 货物运输中 | 司机确认提货 | 查看轨迹、更新状态、异常登记 | 已到达、异常 | 异常可触发异常处理流程 |
| 已到达 | ARRIVED | 货物已到达目的地 | 司机到达目的地 | 签收确认、异常登记 | 已签收、异常 | 客户拒收记录异常 |
| 已签收 | SIGNED | 货物已签收，回单已确认 | 回单确认完成 | 查看详情、回单归档 | 已完成 | 回单异常可重新上传 |
| 已完成 | COMPLETED | 订单全部流程完成 | 回单归档完成 | 查看详情 | - | - |
| 已取消 | CANCELLED | 订单已取消 | 取消操作完成 | 查看详情 | - | 取消后可重新下单 |
| 异常 | EXCEPTION | 订单存在异常情况 | 异常登记/系统预警 | 异常处理、查看详情 | 运输中、已取消、已完成 | 处理完成恢复运输 |

### 7.2 状态流转图

```
待派单 → (智能派单/手动派单) → 已派单 → (司机接单) → 运输中 → (到达目的地) → 已到达 → (签收确认) → 已签收 → (回单归档) → 已完成
                              ↓
                              (取消) → 已取消
                              ↓
                           (异常登记) → 异常 → (处理完成) → 运输中/已取消/已完成
```

## 9. 交互逻辑

### 9.1 页面初始化

- 加载订单基本信息
- 判断当前用户角色和订单状态，控制操作按钮显示
- 加载承运信息（如有）
- 加载运输里程碑时间轴
- 加载回单状态
- 加载操作日志

### 9.2 编辑交互

- 点击编辑按钮，弹出编辑弹窗
- 弹窗内展示可编辑字段（根据状态限制）
- 字段校验规则同新建订单
- 保存后刷新页面数据，提示成功
- 取消编辑不保存数据，关闭弹窗

### 9.3 派单交互

- 智能派单：点击后显示推荐方案弹窗，包含推荐车辆、司机、预计到达时间、匹配度评分
- 手动派单：点击后显示选择器，筛选可用车辆和司机
- 两种派单方式都需要确认派单二次确认
- 派单成功后自动刷新承运信息区域

### 9.4 取消交互

- 点击取消按钮，弹出取消确认框
- 提供取消原因下拉选择（客户取消、货物异常、无法派送、其他）
- 如订单已有运输任务，提示需要先处理车辆释放
- 取消成功后更新订单状态，隐藏派单相关操作

### 9.5 轨迹查看交互

- 点击查看轨迹，跳转至运输轨迹页面（传递订单ID参数）
- 运输轨迹页面自动筛选该订单的运输任务
- 支持轨迹回放和里程碑查看

### 9.6 详情与跳转

- 订单号点击：复制订单号到剪贴板
- 客户名称点击：跳转到客户详情（如果有）
- 车牌号点击：跳转到车辆详情
- 司机姓名点击：跳转到司机详情
- 返回按钮：返回订单列表（记住列表筛选状态）

## 15. 原型生成说明

### 15.1 页面布局建议

- **整体布局**：标准详情页布局，左侧为订单信息卡片区域（70%宽度），右侧为操作日志和快捷操作区域（30%宽度）
- **顶部区域**：订单号+状态Tag占据一行，右側放置操作按钮组
- **信息卡片**：使用Ant Design Card组件，每个分组一个卡片，可折叠
- **时间轴**：使用Ant Design Timeline组件，垂直布局
- **弹窗**：使用Ant Design Modal组件，居中显示，宽度600-800px

### 15.2 关键交互优先级

1. **P0-必须**：订单详情信息完整展示、状态Tag颜色、进度Steps
2. **P0-必须**：编辑、取消、派单、复制操作按钮及弹窗
3. **P0-必须**：承运信息显示（已派单订单）
4. **P0-必须**：运输里程碑时间轴
5. **P1-重要**：回单上传与确认功能
6. **P1-重要**：操作日志时间轴
7. **P2-可选**：地图预览小控件

### 15.3 需要重点表达的业务逻辑

1. **状态颜色区分**：不同订单状态使用不同颜色的Tag，一目了然
2. **操作权限控制**：根据用户角色和订单状态动态显示可用操作按钮
3. **流程进度可视化**：Steps组件展示订单当前处于哪个阶段
4. **时间轴追溯**：运输里程碑记录每个节点的时间和状态
5. **派单闭环**：智能派单和手动派单的完整交互流程
