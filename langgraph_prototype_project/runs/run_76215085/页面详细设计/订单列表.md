# 订单列表页面详细设计

## 1. 页面基础信息

| 字段 | 内容 |
|------|------|
| 页面名称 | 订单列表 |
| 页面编号 | P02 |
| 所属模块 | 订单管理 |
| 页面类型 | 列表页 |
| 路由路径 | /order/list |
| 访问角色 | 调度员、客服、仓库管理员 |
| 关联功能点 | F101（订单列表展示）、F106（订单多条件筛选）、F107（订单数据导出） |
| 关联业务场景 | 场景1：订单全生命周期管理 |
| 关联数据对象 | 订单 |

## 2. 页面目标与使用场景

### 2.1 页面目标

- 展示系统中所有物流订单的列表数据
- 支持多条件组合筛选，快速定位目标订单
- 提供订单导出功能，支持数据离线分析
- 作为订单管理的核心入口，承载订单的查看、编辑、派单等操作

### 2.2 使用场景

| 场景 | 描述 | 触发角色 |
|------|------|----------|
| 日常订单查询 | 客服或调度员日常查询订单状态 | 调度员、客服 |
| 订单筛选定位 | 根据客户、线路、时间等条件筛选订单 | 调度员、客服 |
| 订单详情查看 | 点击订单查看完整信息 | 调度员、客服、仓库 |
| 订单导出 | 导出订单数据用于统计分析 | 调度员、客服、管理人员 |
| 新建订单入口 | 从列表页进入新建订单页面 | 客服 |
| 快速派单 | 从列表页快速进入调度页面 | 调度员 |

### 2.3 触发入口

- 左侧菜单：订单管理 → 订单列表
- 首页快捷操作：新建订单
- 待办事项：待处理订单跳转

### 2.4 输出结果

- 展示订单列表数据（默认20条/页）
- 筛选后可更新列表数据
- 导出生成Excel文件
- 点击订单可跳转到订单详情页面

## 7. 业务状态与状态流转

### 7.1 订单状态流转图

```
创建订单 → PENDING（待派单）
              ↓
         DISPATCHED（已派单）
              ↓
         IN_TRANSIT（运输中）
              ↓
          ARRIVED（已到达）
              ↓
        COMPLETED（已完成）
              ↓
          回单归档
```

### 7.2 状态流转说明

| 当前状态 | 可执行操作 | 下一状态 | 触发条件 |
|----------|------------|----------|----------|
| PENDING | 派单、取消 | DISPATCHED / CANCELLED | 调度员派单/客服取消 |
| DISPATCHED | 开始运输、取消 | IN_TRANSIT / CANCELLED | 司机确认接单/调度员取消 |
| IN_TRANSIT | 更新状态、异常登记 | IN_TRANSIT / ARRIVED / EXCEPTION | 司机更新位置/到达目的地/发生异常 |
| ARRIVED | 签收确认 | COMPLETED | 仓库/收货人确认签收 |
| COMPLETED | 回单归档 | 回单归档 | 客服确认回单 |
| CANCELLED | 无 | - | 订单终止 |
| EXCEPTION | 异常处理 | IN_TRANSIT / CANCELLED | 异常处理完成 |

### 7.3 状态相关操作权限

## 9. 交互逻辑

### 9.1 页面初始化

- 默认展示：全部订单列表，按创建时间倒序排列
- 默认筛选：无（展示全部订单）
- 默认排序：按创建时间降序（最新在前）
- 默认分页：第1页，每页20条
- 初始化异常：如果数据加载失败，显示错误提示和重试按钮

### 9.2 查询与筛选

#### 筛选规则

| 筛选项 | 筛选方式 | 逻辑说明 |
|--------|----------|----------|
| 客户 | 下拉单选 | 精确匹配customerId |
| 订单状态 | 下拉单选 | 精确匹配orderStatus，多选时使用IN查询 |
| 时间范围 | 日期范围 | 精确匹配createTime在范围内 |
| 线路关键词 | 文本输入 | 模糊匹配shipperCity或consigneeCity |
| 订单号 | 文本输入 | 精确或模糊匹配orderNo |

#### 筛选交互

1. 用户填写筛选条件后，点击"查询"按钮
2. 系统验证必填项（如时间范围起止日期）
3. 验证通过后，发送筛选请求
4. 请求返回后更新表格数据
5. 如果无符合条件数据，显示空状态提示

#### 重置规则

- 点击"重置"按钮后，清空所有筛选条件
- 自动触发一次查询，加载全部数据
- 重置后分页恢复为第1页

#### 无结果状态

- 展示空状态插图和文案"暂无符合筛选条件的订单"
- 提供"重置筛选"按钮，用户可快速清空条件

- 点击行：打开订单详情抽屉
- 双击行：无特殊处理

### 9.4 详情与跳转

#### 详情入口

- 点击表格行任意位置（除操作列按钮外）
- 点击操作列的"查看"按钮
- 点击待办事项中的订单链接

#### 跳转目标

- 订单详情：打开右侧抽屉，展示订单完整信息
- 智能调度：点击"调度"按钮，跳转到智能调度页面，并自动选中该订单

#### 返回规则

- 从订单详情抽屉返回：关闭抽屉，停留在当前列表页
- 从列表页返回（如有面包屑）：保持当前筛选和分页状态

#### 面包屑规则

- 面包屑路径：首页 > 订单管理 > 订单列表
- 点击"订单管理"返回订单列表（保持当前筛选状态）
- 点击"首页"返回首页

## 12. 业务流程设计

### 12.1 订单查询流程

```
进入订单列表 → 初始化加载全部订单 → 用户填写筛选条件 → 
点击查询 → 系统验证筛选条件 → 查询数据库 → 返回结果 → 
更新表格展示 → 用户查看/操作订单
```

### 12.2 订单派单流程（从列表页发起）

```
订单列表 → 点击待派单订单的"调度"按钮 → 
跳转智能调度页面 → 自动选中该订单 → 
调度员进行派单操作 → 订单状态更新为已派单 → 
返回订单列表刷新数据
```

### 12.3 订单取消流程

```
订单列表 → 点击"取消"按钮 → 弹出取消原因填写弹窗 → 
选择取消原因并填写备注 → 点击确认 → 
系统校验权限 → 更新订单状态为已取消 → 
刷新列表数据 → 提示"订单已取消"
```

### 12.4 订单导出流程

```
点击导出按钮 → 弹出导出选项（当前筛选/全部）→ 
用户选择范围 → 点击导出 → 
系统生成Excel文件 → 浏览器下载 → 提示"导出成功"
```

## 16. 原型生成说明

### 16.1 页面布局建议

- 整体布局：标准管理后台布局，顶部页面标题+操作按钮区，筛选区在标题下方，表格区域占主要空间，底部固定分页
- 页面宽度：最小支持1200px，最大支持1920px，超出部分横向滚动
- 响应式：桌面端优先，移动端简化筛选区域为折叠状态

### 16.2 关键交互优先级

1. **P0-必须交互**：列表加载、筛选查询、分页切换、查看详情、派单跳转
2. **P1-重要交互**：取消订单、订单状态筛选、重置筛选
3. **P2-一般交互**：导出功能、排序操作、每页条数切换

### 16.3 需要重点表达的业务逻辑

- 订单状态的颜色标识和文字说明
- 不同角色看到不同的操作按钮（调度员有派单、客服有新建和取消、仓库只有查看）
- 筛选条件与表格数据的联动
- 派单跳转时订单的自动选中状态

### 16.4 可使用模拟数据

提供20+条模拟订单数据，包含以下状态分布：

| 状态 | 数量 | 说明 |
|------|------|------|
| 待派单 | 5条 | 可进行派单操作 |
| 已派单 | 3条 | 已分配车辆和司机 |
| 运输中 | 8条 | 正在运输中 |
| 已到达 | 2条 | 已到达目的地等待签收 |
| 已完成 | 10条 | 已完成订单 |
| 已取消 | 2条 | 已取消订单 |
| 异常 | 2条 | 运输异常订单 |

模拟数据示例字段：
- 订单号：ORD202401010001 ~ ORD202401010025
- 客户：北京某某公司、上海某某公司、广州某某公司等
- 线路：北京-上海、广州-深圳、成都-重庆等
- 货物：电子产品、服装、建材、食品等
- 时效：24小时、48小时、72小时等

### 16.5 需要避免的原型误差

- 禁止只展示空白的表格而无数据
- 禁止筛选区域只有UI而无交互功能
- 禁止分页器只展示UI而无切换功能
- 禁止点击订单详情无任何反应
- 禁止状态标签不区分颜色或颜色错误
- 禁止操作按钮根据状态不动态显示
- 禁止导出按钮点击后无任何反馈
