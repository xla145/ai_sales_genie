# 运输轨迹页面详细设计

## 1. 页面基础信息

| 字段 | 内容 |
|------|------|

| 页面编号 | P12 |

| 路由路径 | /track/index |

| 关联数据对象 | 车辆、轨迹记录、运输任务 |

## 2. 页面目标与使用场景

### 页面目标

1. 实时展示运输中车辆的位置信息，在地图上可视化呈现
2. 支持查看历史运输轨迹并进行回放演示
3. 展示运输过程中的里程碑节点（装车、发车、途径、到达等）
4. 支持手动更新运输状态
5. 提供多条件查询和快速筛选功能

### 使用场景

| 场景 | 描述 | 触发条件 |
|------|------|----------|
| 实时监控 | 调度员监控运输中车辆当前位置 | 运输任务进行中 |
| 轨迹回放 | 查看已完成运输的历史轨迹 | 需要回顾运输过程 |
| 客户咨询 | 客服人员回答客户关于货物位置的问题 | 客户来电咨询 |
| 异常排查 | 管理人员排查运输异常原因 | 发现运输异常 |
| 状态更新 | 司机或调度员手动更新运输状态 | 运输节点变化 |

### 触发入口

1. 顶部导航菜单：运输轨迹
2. 首页快捷操作：点击"运输追踪"
3. 订单详情页：点击"查看轨迹"按钮
4. 智能调度页：点击车辆标记

### 输出结果

1. 地图展示车辆实时位置和运输路线
2. 里程碑时间轴展示运输节点
3. 轨迹数据可用于回放和分析

## 7. 业务状态与状态流转

### 7.1 运输任务状态

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |
|----------|----------|----------|----------|------------|----------|----------|
| 已装车 | LOADED | 货物已装车，等待出发 | 仓库确认装车 | 更新状态、查看轨迹 | 运输中 | 无 |
| 运输中 | IN_TRANSIT | 正在运输途中 | 司机确认发车 | 更新状态、查看轨迹、轨迹回放 | 已到达 | 可上报异常 |
| 已到达 | ARRIVED | 到达目的地 | 车辆到达目的地 | 确认到货、查看轨迹 | 卸货中 | 可上报异常 |
| 卸货中 | UNLOADING | 正在卸货 | 開始卸货 | 更新状态 | 已卸货 | 无 |
| 已卸货 | UNLOADED | 卸货完成 | 卸货完成 | 签收确认 | 待签收 | 可上报异常 |
| 已签收 | SIGNED | 已完成签收 | 客户确认签收 | 查看回单 | 已完成 | 无 |
| 已完成 | COMPLETED | 运输任务完成 | 全部流程完成 | 查看详情、轨迹回放 | - | 无 |
| 异常中止 | EXCEPTION | 运输异常终止 | 发生异常 | 异常处理 | 异常处理 | 恢复/取消 |

### 7.2 状态流转图

```
已装车 → 运输中 → 已到达 → 卸货中 → 已卸货 → 已签收 → 已完成
                       ↓                      ↓
                    异常中止 ←←←←←←←←←←←←←←←←←← 异常中止
```

### 7.3 手动状态更新

| 当前状态 | 可更新为 | 操作权限 | 前置条件 | 备注 |
|----------|----------|----------|----------|------|
| 已装车 | 运输中 | 司机、调度员 | 已装车确认 | 触发发车 |
| 运输中 | 已到达 | 司机、调度员 | 到达目的地 | 触发到达 |
| 运输中 | 异常中止 | 司机、调度员 | 发生异常 | 触发异常流程 |
| 已到达 | 卸货中 | 仓库、司机 | 开始卸货 | - |

| 已卸货 | 已签收 | 客服、仓库 | 客户签收 | 触发签收 |

## 9. 交互逻辑

### 9.1 页面初始化

1. **默认展示**：
   - 地图加载默认中心点（可配置，默认城市中心）
   - 左侧任务列表默认加载"运输中"状态的任务
   - 底部时间轴默认收起

2. **默认筛选**：
   - 状态筛选默认选择"运输中"
   - 时间范围默认显示当天

3. **默认排序**：
   - 任务列表按预计到达时间升序排列

4. **初始化异常**：
   - 地图加载失败：显示地图加载失败提示，提供重试按钮
   - 数据加载失败：显示数据加载失败提示，提供重试按钮

### 9.2 查询与筛选

| 筛选项 | 类型 | 选项/格式 | 筛选规则 |
|--------|------|-----------|----------|
| 订单筛选 | 下拉选择 | 订单号搜索/选择 | 模糊匹配订单号 |
| 车辆筛选 | 下拉选择 | 车牌号搜索/选择 | 模糊匹配车牌号 |
| 时间范围 | 日期范围 | YYYY-MM-DD | 筛选创建时间 |
| 运输状态 | 标签按钮 | 全部/运输中/已完成/异常 | 精确匹配状态 |
| 快速筛选 | 按钮 | 我的任务/本区域任务 | 按角色数据范围筛选 |

- **查询规则**：点击"查询"按钮后执行筛选，支持组合筛选
- **重置规则**：点击"重置"按钮清空所有筛选条件
- **无结果状态**：显示"暂无符合条件的运输任务"空状态插图
- **查询异常**：显示错误提示，提供重试按钮

### 9.3 地图交互

1. **缩放控制**：鼠标滚轮/点击缩放按钮，支持缩放到车辆级别
2. **平移拖拽**：鼠标拖拽移动地图视野
3. **点击标记**：点击车辆标记弹出信息窗，显示车辆和任务简要信息
4. **点击轨迹**：鼠标悬停轨迹线显示该段轨迹信息
5. **点击里程碑**：点击里程碑标记弹出节点详情
6. **视野自适应**：点击"查看全部"按钮自动调整视野包含所有车辆

### 9.4 任务列表交互

1. **点击选中**：点击任务项选中，高亮显示，地图定位到该车辆
2. **双击详情**：双击任务项打开详情面板
3. **右键菜单**：右键点击任务弹出操作菜单（查看详情、轨迹回放等）
4. **拖拽排序**：支持拖拽调整任务顺序

### 9.5 轨迹回放交互

1. **播放控制**：播放/暂停/停止按钮
2. **进度控制**：拖拽进度条选择回放时间点
3. **速度控制**：支持1x/2x/5x/10x倍速
4. **时间显示**：显示当前回放时间点

## 12. 异常、空状态和边界场景

| 场景 | 触发条件 | 页面表现 | 用户操作 | 恢复方式 |

## 15. 原型生成说明

### 页面布局建议

1. **整体布局**：采用左右分栏+底部时间轴布局
   - 左侧任务列表：宽度280px，可收起
   - 中部地图区域：自适应剩余宽度
   - 右侧详情面板：宽度360px，可收起
   - 底部时间轴：高度120px，可收起

2. **地图区域**：占据页面主要空间，使用高德地图或百度地图SDK

3. **筛选工具栏**：固定在顶部，高度56px

### 关键交互优先级

| 优先级 | 交互 | 说明 |
|--------|------|------|
| P0 | 实时位置展示 | 地图上显示车辆位置，实时刷新 |
| P0 | 任务列表展示 | 运输任务列表，支持筛选和选中 |
| P0 | 地图与任务联动 | 选中任务后地图定位到车辆 |
| P1 | 轨迹回放 | 历史轨迹动画回放 |
| P1 | 状态更新 | 手动更新运输状态 |
| P1 | 里程碑时间轴 | 运输节点时间轴展示 |
| P2 | 轨迹导出 | 导出轨迹数据 |

### 需要重点表达的业务逻辑

1. **实时追踪**：车辆标记在地图上移动，模拟实时定位效果
2. **轨迹可视化**：已行驶路线用轨迹线表示，未行驶路线用虚线表示
3. **里程碑节点**：装车、发车、途径、到达等节点在轨迹上标记
4. **状态流转**：不同状态用不同颜色标签区分
5. **车辆联动**：选中任务列表项，地图自动定位到对应车辆

### 可使用模拟数据

1. **运输任务**：至少5条模拟任务，包含不同状态（运输中、已完成等）
2. **轨迹数据**：1条完整运输轨迹，包含10+个轨迹点
3. **里程碑**：5+个里程碑节点（已装车、运输中、途径点、已到达、已签收）
4. **车辆位置**：模拟车辆在运输路线上的实时位置

### 需要避免的原型误差

1. 避免地图区域为空白占位，必须有地图底图和交互
2. 避免任务列表无数据，必须有模拟任务展示
3. 避免轨迹为静态线条，必须有回放动画演示
4. 避免状态更新无反馈，必须有成功/失败提示
5. 避免里程碑时间轴为空，必须有时间轴节点展示
