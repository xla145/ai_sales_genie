## 2. 系统边界与业务闭环

### 2.1 系统边界

**系统负责范围：**
- 订单全生命周期管理（新建、派单、运输、签收、归档）
- 车辆信息管理与状态监控
- 司机信息管理与排班
- 智能调度与派单
- 运输轨迹实时追踪与回放
- 异常预警与处理
- 签收回单管理
- 数据统计与分析
- 系统配置与权限管理

**不包含范围：**
- 财务结算与支付功能（仅展示统计数据）
- ERP/WMS等外部系统对接（预留接口）
- 移动端App开发（首期仅Web端）
- 硬件GPS设备研发（仅对接）
- 实际货物装卸与仓储管理

### 2.2 核心业务闭环

**闭环1：订单履约闭环**
```
客户发货需求 → 新建订单 → 订单审核 → 智能派单 → 司机接单 → 
运输执行 → 轨迹追踪 → 到达目的地 → 签收回单 → 订单归档
```

**闭环2：调度优化闭环**
```
待派单订单 → 智能算法推荐 → 调度员确认 → 车辆分配 → 
司机接收任务 → 路线规划 → 执行运输
```

**闭环3：异常处理闭环**
```
异常发生（自动预警/人工上报） → 异常登记 → 评估定级 → 
处理方案制定 → 处理执行 → 处理确认 → 归档改进
```

### 2.3 核心业务对象

| 业务对象 | 说明 | 关键属性 |
| --- | --- | --- |
| 订单 | 物流运输需求 | 订单号、客户、发货地、目的地、货物信息、时效要求、订单状态 |
| 车辆 | 运输工具 | 车牌号、车型、载重、GPS设备、状态、当前位置 |

## 4. 全局业务场景

### 场景1：订单全生命周期管理

- **场景目标**：实现订单从创建到归档的完整流程管理
- **参与角色**：客服人员、调度员、司机、仓库管理员
- **触发条件**：客户发货需求
- **前置条件**：客户信息已维护、车辆和司机资源可用
- **核心流程**：
  1. 客服/客户新建订单，填写发货信息
  2. 调度员审核订单，进行智能派单或手动派单
  3. 司机接单，确认接单并开始运输
  4. 运输过程中实时更新轨迹状态
  5. 到达目的地，仓库/收货人确认收货
  6. 司机提交签收回单，订单完成归档
- **分支流程**：订单取消、派单调整、异常中断
- **异常流程**：派单失败、车辆故障、无法签收
- **涉及功能**：订单列表、订单详情、新建订单、智能调度、运输轨迹、签收回单
- **涉及页面**：订单列表、订单详情、新建订单、调度看板、运输轨迹
- **涉及数据对象**：订单、运输任务、回单
- **输出结果**：订单状态更新为已完成，回单归档
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景2：智能调度与路线规划

- **场景目标**：通过智能算法实现最优派单和路线规划
- **参与角色**：调度员、管理人员
- **触发条件**：有待派单订单
- **前置条件**：订单已确认、车辆和司机信息完备
- **核心流程**：
  1. 调度员查看待派单订单列表
  2. 系统根据订单和车辆信息生成智能派单建议
  3. 调度员确认或调整派单方案
  4. 系统生成最优运输路线
  5. 司机收到运输任务通知
- **分支流程**：手动派单、批量派单、派单调整
- **异常流程**：无可用车辆、派单冲突、算法异常
- **涉及功能**：智能派单、手动派单、路线规划、派单调整
- **涉及页面**：智能调度、订单详情
- **涉及数据对象**：订单、车辆、司机、运输任务
- **输出结果**：派单成功，生成运输任务和路线
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景3：运输实时追踪与状态更新

- **场景目标**：实时掌握运输状态和车辆位置
- **参与角色**：调度员、客服人员、管理人员、客户
- **触发条件**：运输任务开始
- **前置条件**：车辆已安装定位设备、系统可接收定位数据
- **核心流程**：
  1. 司机开始运输，系统记录出发状态
  2. 车辆定位实时更新，显示在轨迹地图上
  3. 系统自动更新运输里程碑节点
  4. 异常情况自动预警提醒
  5. 到达目的地，自动提醒签收
- **分支流程**：轨迹回放、状态手动更新
- **异常流程**：GPS离线、定位延迟、状态更新失败
- **涉及功能**：实时追踪、轨迹回放、里程碑记录、状态更新
- **涉及页面**：运输轨迹
- **涉及数据对象**：运输任务、轨迹记录
- **输出结果**：实时运输轨迹、状态更新记录
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景4：异常预警与处理

- **场景目标**：快速发现异常并完成处理闭环
- **参与角色**：司机、调度员、客服人员、管理人员
- **触发条件**：发生异常事件（延迟、路线偏离、货物异常、车辆故障等）
- **前置条件**：异常监控规则已配置
- **核心流程**：
  1. 系统检测到异常或司机上报异常
  2. 系统生成异常预警，推送给相关人员
  3. 调度员评估异常情况
  4. 制定异常处理方案并执行
  5. 处理结果记录归档
  6. 如需赔偿则触发后续流程
- **分支流程**：异常升级、多级审批、赔偿处理
- **异常流程**：异常未处理、重复异常、处理超时
- **涉及功能**：异常列表、异常详情、异常登记、异常处理、异常预警、异常统计
- **涉及页面**：异常处理、异常详情
- **涉及数据对象**：异常记录、运输任务
- **输出结果**：异常处理记录、改进措施
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景5：数据统计与决策支持

- **场景目标**：提供全面的运营数据统计和分析
- **参与角色**：管理人员、财务人员
- **触发条件**：日常运营管理需要
- **前置条件**：数据采集完备
- **核心流程**：
  1. 管理人员登录系统
  2. 查看数据看板主页
  3. 查看各维度统计报表
  4. 分析异常数据或趋势
  5. 制定改进措施
- **分支流程**：报表导出、自定义报表
- **异常流程**：数据缺失、统计口径不一致
- **涉及功能**：运营概览、订单统计、车辆统计、司机统计、财务统计、报表导出
- **涉及页面**：数据看板
- **涉及数据对象**：订单、车辆、司机、运输任务、异常记录
- **输出结果**：数据报表、分析结论、决策建议
- **闭环检查**：完整
- **来源/依据**：用户明确

### 6.3 禁止事项

1. 禁止使用"功能开发中"或"页面建设中"等空白占位
2. 禁止只展示静态UI而无交互功能
3. 禁止列表页无筛选和分页功能
4. 禁止表单页无校验和提交反馈
5. 禁止详情页无关联操作入口
6. 禁止弹窗内容为空或无可用操作
7. 禁止异常场景无提示或无空状态处理
