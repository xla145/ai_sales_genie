## 4. 全局业务场景

### 场景 1：订单全流程管理

- **场景目标**：实现从订单创建到签收回单的完整物流运输闭环
- **参与角色**：运营人员、调度员、司机
- **触发条件**：客户下单或运营人员手动创建订单
- **前置条件**：车辆和司机资源可用
- **核心流程**：
  1. 运营人员新建订单，填写发货地、收货地、货物信息、期望送达时间等
  2. 调度员进行智能派单或手动指派车辆和司机
  3. 司机接单并执行运输任务
  4. 运输过程中实时更新轨迹状态
  5. 司机到达目的地，客户签收回单
  6. 订单完成，生成运输数据
- **分支流程**：订单取消、订单修改、派单调整
- **异常流程**：异常上报、订单挂起、重新派单
- **涉及功能**：订单列表、订单详情、新建订单、智能调度、签收回单、运输轨迹
- **涉及页面**：订单列表页、订单详情页、新建订单页、智能调度页、运输轨迹页
- **涉及数据对象**：订单、车辆、司机、运输任务
- **输出结果**：订单状态更新为已完成，回单信息归档
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景 2：智能调度与路线规划

- **场景目标**：高效完成订单分配，降低调度工作量，优化运输效率
- **参与角色**：调度员、系统
- **触发条件**：有待派单订单或调度员主动发起调度
- **前置条件**：订单已录入、车辆和司机信息完整
- **核心流程**：
  1. 系统收集待派单订单和可用资源
  2. 智能算法计算最优分配方案
  3. 调度员查看推荐方案，可调整车辆和司机
  4. 确认派单，通知司机
  5. 司机收到任务提醒
- **分支流程**：手动派单、调度调整、批量派单
- **异常流程**：无可用资源提示、派单失败重试
- **涉及功能**：智能派单、路线规划、手动派单、司机通知
- **涉及页面**：智能调度页、订单详情页
- **涉及数据对象**：订单、车辆、司机、运输任务
- **输出结果**：派单结果推送至司机端
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景 3：运输实时监控

- **场景目标**：实现运输过程可视化，实时掌握货物位置和状态
- **参与角色**：管理人员、运营人员、调度员
- **触发条件**：有运输中的订单
- **前置条件**：车辆安装定位设备并开启定位（模拟）
- **核心流程**：
  1. 系统实时接收车辆定位数据
  2. 在地图上展示车辆位置和运输轨迹
  3. 监测运输状态，发现异常自动预警
  4. 相关人员收到预警通知
- **分支流程**：轨迹回放、多车监控
- **异常流程**：定位信号丢失、轨迹偏离预警
- **涉及功能**：运输轨迹、实时定位、轨迹回放、异常预警
- **涉及页面**：运输轨迹页、订单详情页
- **涉及数据对象**：车辆、运输任务、轨迹点
- **输出结果**：实时轨迹展示、异常预警推送
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景 4：异常处理

- **场景目标**：快速发现和处理运输异常，降低损失和风险
- **参与角色**：司机、调度员、管理人员
- **触发条件**：异常事件发生或系统检测到异常
- **前置条件**：有运输中的订单
- **核心流程**：
  1. 司机上报异常或系统自动检测异常
  2. 系统生成异常单，记录异常类型、详情
  3. 调度员或管理人员收到异常通知
  4. 处理异常，记录处理方案和结果
  5. 异常恢复，订单继续执行或终止
- **分支流程**：异常升级、异常取消、重新派单
- **异常流程**：异常处理超时、异常升级管理层
- **涉及功能**：异常上报、异常预警、异常处理、异常记录
- **涉及页面**：异常处理页、异常列表页、异常详情页
- **涉及数据对象**：异常、订单、运输任务
- **输出结果**：异常处理记录、订单状态更新
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景 5：数据统计分析

- **场景目标**：为管理决策提供数据支撑，掌握运营状况
- **参与角色**：管理人员
- **触发条件**：需要查看数据时
- **前置条件**：有历史运营数据
- **核心流程**：
  1. 选择统计维度（订单、车辆、司机、收入等）
  2. 设置时间范围和其他筛选条件
  3. 系统生成统计报表和图表
  4. 管理人员查看分析
- **分支流程**：自定义报表、数据导出
- **异常流程**：数据加载失败、查询超时
- **涉及功能**：订单统计、运输统计、收入统计、车辆统计、司机统计
- **涉及页面**：数据看板页
- **涉及数据对象**：订单、车辆、司机、运输任务
- **输出结果**：数据报表、图表展示
- **闭环检查**：完整
- **来源/依据**：用户明确

### 场景 6：系统管理

- **场景目标**：维护系统正常运行，管理用户和权限
- **参与角色**：系统管理员
- **触发条件**：系统管理需求
- **前置条件**：系统管理员账号
- **核心流程**：
  1. 管理系统用户账号
  2. 配置角色权限
  3. 管理组织架构
  4. 配置系统参数
- **涉及功能**：用户管理、角色权限、组织架构、系统参数
- **涉及页面**：系统设置页
- **涉及数据对象**：用户、角色、权限、组织
- **输出结果**：系统配置更新
- **闭环检查**：完整
- **来源/依据**：用户明确

## 5. 全局设计规范

### 5.1 路由命名规则

| 规则项 | 规则说明 | 示例 |
| --- | --- | --- | 
| 路径格式 | 使用小写字母和短横线 | /order-list、/vehicle-management |
| 模块前缀 | 按模块划分路由前缀 | /order/*、/vehicle/*、/driver/* |
| 页面命名 | 使用有意义的页面标识符 | /order-create、/order-detail、/dispatch-center |
| 参数传递 | 路径参数使用冒号标识 | /order-detail/:id、/vehicle-detail/:id |
| 命名原则 | 简洁明了、与菜单名称对应 | dispatch 对应 智能调度 |
