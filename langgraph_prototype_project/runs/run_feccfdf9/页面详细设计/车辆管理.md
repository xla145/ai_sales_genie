# 车辆管理页面详细设计

## 1. 页面基础信息

| 字段 | 内容 |
|------|------|
| 页面名称 | 车辆管理 |
| 页面编号 | P05 |
| 所属模块 | 车辆管理 |
| 页面类型 | 列表页 |
| 路由路径 | /vehicle/list |
| 访问角色 | 调度员、车辆管理人员 |
| 关联功能点 | F02-01（车辆列表展示）、F02-06（车辆删除） |
| 关联业务场景 | 智能调度、车辆信息维护 |
| 关联数据对象 | 车辆 |

## 2. 页面目标与使用场景

### 2.1 页面目标

- 展示所有车辆信息，支持多维度筛选和搜索
- 提供车辆状态实时监控
- 支持新增、编辑、删除车辆操作
- 为智能调度提供可用车辆资源

### 2.2 使用场景

| 场景 | 触发条件 | 用户操作 | 预期结果 |
|------|----------|----------|----------|
| 车辆列表查看 | 用户进入车辆管理页面 | 页面加载，展示车辆列表 | 默认展示全部车辆，按更新时间倒序排列 |
| 车辆筛选 | 用户选择筛选条件 | 选择车辆状态、车型等筛选条件 | 列表更新为符合筛选条件的车辆 |
| 车辆搜索 | 用户输入搜索关键词 | 在搜索框输入车牌号、车型等关键词 | 列表更新为匹配搜索条件的车辆 |
| 新增车辆 | 用户点击新增按钮 | 点击“新增车辆”按钮 | 跳转到新增车辆页面 |
| 车辆详情 | 用户点击车辆行 | 点击表格中的车辆记录 | 跳转到车辆详情页面 |
| 车辆编辑 | 用户点击编辑按钮 | 点击操作列的编辑按钮 | 跳转到编辑车辆页面 |
| 车辆删除 | 用户点击删除按钮 | 点击操作列的删除按钮 | 弹出确认框，确认后删除车辆 |

### 2.3 触发入口

- 左侧菜单：车辆管理 → 车辆管理
- 首页快捷入口：车辆管理
- 智能调度页：点击可用车辆列表项

### 2.4 输出结果

- 车辆列表数据展示
- 筛选后的车辆数据
- 操作结果提示（新增成功、删除成功等）
- 页面跳转至详情页/编辑页

## 7. 业务状态与状态流转

### 7.1 页面业务状态

| 状态名称 | 状态说明 | 触发条件 |
|----------|----------|----------|
| 正常 | 列表正常展示 | 页面加载成功 |
| 加载中 | 表格数据加载中 | 接口请求中 |
| 空数据 | 无车辆数据 | 查询结果为空 |
| 查询无结果 | 筛选条件无匹配 | 筛选后无结果 |

### 7.2 车辆状态流转

| 当前状态 | 触发事件 | 下一状态 | 处理逻辑 |
|----------|----------|----------|----------|
| 空闲 | 派单成功 | 使用中 | 更新车辆状态和当前司机 |
| 使用中 | 任务完成 | 空闲 | 更新车辆状态，清除当前司机 |
| 空闲 | 开始维保 | 维修中 | 更新车辆状态，记录维保开始时间 |

| 任意状态 | 定位设备离线 | 离线 | 自动检测，更新状态 |
| 离线 | 定位设备上线 | 空闲/使用中 | 自动检测，根据实际任务更新 |

### 7.3 状态筛选对应关系

| 筛选值 | 对应状态 | 说明 |
|--------|----------|------|

## 9. 交互逻辑

### 9.1 页面初始化

- 默认加载全部车辆数据
- 默认按更新时间倒序排列
- 默认每页显示10条数据
- 默认筛选条件为空（显示全部）
- 如果有维保提醒，在页面顶部显示提醒横幅

### 9.2 查询与筛选

#### 9.2.1 搜索功能

- 搜索字段：车牌号（模糊匹配）、车型（精确匹配）、所属公司（模糊匹配）
- 搜索触发：回车键、搜索按钮点击
- 搜索延迟：输入完成后300ms触发搜索
- 搜索时清除分页，回到第一页

#### 9.2.2 筛选功能

- 状态筛选：单选，选择后立即触发筛选
- 车型筛选：单选，选择后立即触发筛选
- 筛选支持组合：状态+车型+搜索关键词
- 筛选时清除分页，回到第一页
- 筛选条件有值时显示“已筛选”标记

#### 9.2.3 清除功能

- 点击清除按钮重置所有筛选条件
- 重置后自动刷新列表，显示全部数据
- 清除按钮仅在有筛选条件时显示

### 9.3 分页逻辑

- 默认每页10条，支持切换20条/50条
- 切换页码后滚动到表格顶部
- 总数少于一页时隐藏分页组件
- 分页信息展示：共X条记录，第X/Y页

### 9.4 表格操作

- 点击表格行（非操作列）跳转车辆详情
- 排序：点击表头可按列排序（默认按更新时间倒序）
- 状态列显示状态标签，颜色按状态颜色规则
- 操作列固定在右侧，包含编辑和删除按钮

### 9.5 删除操作

- 点击删除按钮弹出确认框
- 确认框显示车辆信息和删除提示
- 确认后发送删除请求
- 删除成功后显示成功提示，刷新列表
- 删除失败显示错误提示，列表不变
- 车辆状态为使用中或维修中时，禁用删除按钮

### 9.6 新增/编辑跳转

- 新增按钮点击后跳转到 /vehicle/create
- 编辑按钮点击后跳转到 /vehicle/edit/:id
- 返回时刷新列表数据

## 11. 业务流程设计

### 11.1 车辆管理流程

```
用户进入车辆管理页
    ↓
页面加载车辆列表（默认全部，按更新时间倒序）
    ↓
用户可进行以下操作：
├── 搜索（按车牌号/公司名称）
├── 筛选（按状态/车型）
├── 新增车辆 → 跳转新增页面
├── 查看详情 → 点击表格行 → 跳转详情页
├── 编辑车辆 → 点击编辑 → 跳转编辑页
└── 删除车辆 → 确认 → 删除成功刷新列表
```

### 11.2 车辆状态管理流程

```
车辆状态：空闲
    ↓（调度派单）
车辆状态：使用中（关联订单和司机）
    ↓（任务完成）
车辆状态：空闲
    ↓（发起维保）
车辆状态：维修中
    ↓（维保完成）
车辆状态：空闲
    ↓（定位设备离线检测）
车辆状态：离线
    ↓（定位设备上线检测）
车辆状态：根据实际情况更新
```

## 12. 异常、空状态和边界场景

| 场景 | 触发条件 | 页面表现 | 用户操作 | 恢复方式 |
|------|----------|----------|----------|----------|
| 空数据 | 系统无车辆数据 | 显示空状态插图和提示“暂无车辆信息”，显示新增按钮 | 点击新增按钮创建车辆 | 点击新增按钮 |
| 搜索无结果 | 搜索关键词无匹配 | 显示空状态插图和提示“未找到匹配车辆”，显示清除按钮 | 点击清除按钮 | 点击清除按钮 |
| 筛选无结果 | 筛选条件无匹配 | 显示空状态插图和提示“暂无符合条件的车辆”，显示清除按钮 | 点击清除按钮 | 点击清除按钮 |
| 加载失败 | 接口请求失败 | 显示错误提示和重试按钮 | 点击重试按钮 | 点击重试按钮重新请求 |

## 15. 原型生成说明

### 15.1 页面布局建议

- 页面采用标准的的管理后台布局
- 顶部：页面标题（左侧）+ 新增按钮（右侧）
- 筛选区域：搜索框 + 状态筛选 + 车型筛选 + 清除按钮，一行排列，间距16px
- 内容区域：车辆列表表格，撑满可用高度
- 底部：分页组件，左对齐显示统计信息，右对齐显示分页控件
- 空状态时内容区域显示空状态组件

### 15.2 关键交互优先级

1. **高优先级**：表格数据加载、搜索筛选、分页切换、删除确认
2. **中优先级**：新增跳转、编辑跳转、详情跳转
3. **低优先级**：状态标签样式、按钮hover效果、空状态插图

### 15.3 需要重点表达的业务逻辑

- 车辆状态的颜色区分（空闲-青色、使用中-蓝色、维修中-灰色、离线-灰色）
- 筛选条件的组合逻辑（状态+车型+搜索关键词）
- 删除操作的前置条件判断（使用中/维修中不允许删除）
- 维保提醒的横幅展示

### 15.4 可使用模拟数据

需要准备至少15条模拟车辆数据，包含：
- 不同状态的车辆：空闲5辆、使用中5辆、维修中2辆、离线3辆
- 不同车型：厢式货车4辆、平板车4辆、罐车3辆、冷藏车2辆、其他2辆
- 部分车辆有当前司机，部分没有
- 部分车辆有维保提醒

### 15.5 需要避免的原型误差

- 状态筛选下拉框必须包含“全部”选项
- 删除按钮在使用中/维修中状态下必须禁用
- 筛选条件有值时必须显示清除按钮
- 空状态必须显示空状态插图，不能留白
- 加载状态必须显示加载动画，不能留白
- 分页组件必须显示总数和当前页信息
