# 订单列表页面详细设计

## 1. 页面基础信息

| 字段 | 内容 |
|------|------|
| 页面名称 | 订单列表 |
| 页面编号 | P02 |
| 所属模块 | 订单管理 |
| 页面类型 | 列表页 |
| 路由路径 | /order/list |
| 访问角色 | 运营人员、调度员、管理人员 |
| 关联功能点 | F01-01（订单列表展示）、F01-08（订单导入）、F01-09（订单导出） |
| 关联业务场景 | 订单全流程管理 |
| 关联数据对象 | 订单 |

## 2. 页面目标与使用场景

### 2.1 页面目标

- 展示所有物流订单，支持多维度筛选和搜索
- 提供订单导入导出功能，便于批量数据处理
- 支持分页展示，每页可显示10/20/50条记录
- 点击订单可跳转至订单详情页进行查看和操作
- 为调度员提供待派单订单的快速入口

### 2.2 使用场景

| 场景 | 触发条件 | 用户操作 | 预期结果 |
|------|----------|----------|----------|
| 查看订单列表 | 用户进入订单列表页 | 系统默认加载全部订单，显示10条/页 | 展示订单列表，包含订单号、客户名称、发货地、收货地、状态等信息 |
| 订单筛选 | 用户输入筛选条件 | 选择订单状态、时间范围、客户名称等筛选条件 | 列表展示符合筛选条件的订单 |
| 订单搜索 | 用户输入搜索关键词 | 在搜索框输入订单号、客户名称、手机号 | 展示匹配的订单记录 |
| 新建订单 | 用户点击新建按钮 | 点击“新建订单“按钮 | 跳转到新建订单页面 /order/create |
| 订单导入 | 用户点击导入按钮 | 点击“导入“按钮，弹出文件选择框 | 弹出导入弹窗，支持上传Excel文件 |
| 订单导出 | 用户点击导出按钮 | 点击“导出“按钮 | 导出当前筛选条件下的订单数据为Excel |
| 查看订单详情 | 用户点击订单行 | 点击订单列表中的某一行 | 跳转到订单详情页 /order/detail/:id |
| 分页操作 | 用户切换分页 | 点击分页数字或翻页按钮 | 切换到对应页码的订单数据 |

### 2.3 触发入口

- 左侧菜单：订单管理 → 订单列表
- 首页快捷入口：新建订单 → 跳转至订单列表（作为返回列表）
- 其他页面返回：订单详情页点击返回 → 返回订单列表

### 2.4 输出结果

- 订单列表数据展示（包含分页）
- 筛选后的订单数据
- 导入/导出的操作反馈

## 9. 交互逻辑

### 9.1 页面初始化

- **默认展示**：加载全部订单列表，按创建时间倒序排列，每页显示10条
- **默认筛选**：无，默认展示全部状态订单
- **默认排序**：按创建时间倒序（最新在前）
- **默认状态**：默认显示第一页数据
- **初始化异常**：展示加载失败提示，显示“重试”按钮，点击重试重新加载

### 9.2 查询与筛选

#### 筛选项配置

| 筛选项 | 类型 | 选项 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 订单状态 | 下拉选择 | 全部、待派单、已派单、运输中、已到达、已签收、已完成、已取消 | 全部 | 按订单状态筛选 |
| 创建时间 | 日期范围 | 选择开始日期和结束日期 | 空 | 按订单创建时间范围筛选 |
| 期望送达时间 | 日期范围 | 选择开始日期和结束日期 | 空 | 按期望送达时间范围筛选 |

#### 搜索配置

| 搜索方式 | 搜索字段 | 说明 |
|----------|----------|------|
| 模糊搜索 | 订单号、客户名称、发货地址、收货地址 | 支持多字段模糊匹配 |

#### 筛选规则

1. 筛选条件改变后自动触发查询（防抖300ms）
2. 搜索按钮点击后触发精确查询
3. 筛选和搜索条件为“与“关系
4. 筛选条件会保留在URL参数中，支持页面刷新后保持筛选状态

#### 无结果状态

- 当筛选结果为空时，展示空状态插图和提示文案：“暂无符合条件的订单”
- 提供“新建订单“和”重置筛选“快捷操作

### 9.3 导入操作

#### 导入弹窗内容

| 区域 | 内容 |
|------|------|
| 标题 | 导入订单 |
| 说明文字 | 请上传Excel文件，支持.xls和.xlsx格式 |
| 文件上传区 | 拖拽或点击上传文件 |
| 文件模板下载 | 提供导入模板下载链接 |
| 底部按钮 | 取消、确认导入 |

#### 导入流程

1. 用户点击导入按钮，弹出导入弹窗
2. 用户上传Excel文件（支持拖拽或点击）
3. 系统解析Excel文件，校验数据格式
4. 校验通过：显示预览数据，确认后执行导入
5. 校验失败：显示错误信息，提示用户修正
6. 导入成功后：关闭弹窗，刷新列表，显示成功提示
7. 导入失败后：显示失败原因，显示失败条数

### 9.4 导出操作

1. 用户点击导出按钮
2. 系统获取当前筛选条件
3. 生成Excel文件（包含所有符合筛选条件的数据）
4. 浏览器自动下载Excel文件
5. 显示导出成功提示

### 9.5 详情与跳转

- **详情入口**：点击订单表格任意行（除操作列外）
- **跳转目标**：/order/detail/:id（订单详情页）
- **返回规则**：订单详情页点击返回按钮，返回订单列表，并保持之前的筛选状态
- **面包屑规则**：首页 > 订单管理 > 订单列表 > 订单详情

## 11. 业务流程设计

### 11.1 订单列表查询流程

```
用户进入页面
    ↓
初始化加载（调用订单列表API，默认全量数据，第一页，每页10条）
    ↓
展示订单列表
    ↓
用户输入筛选条件 → 自动触发筛选 → 重新调用API → 更新列表
    ↓
用户输入搜索词 → 点击搜索 → 调用搜索API → 更新列表
    ↓
用户点击订单行 → 跳转订单详情页
```

### 11.2 订单导入流程

```
用户点击导入按钮
    ↓
弹出导入弹窗
    ↓
用户上传Excel文件
    ↓
系统解析Excel文件
    ↓
数据校验（格式、必填字段等）
    ↓
├─ 校验通过 → 显示预览数据 → 用户确认 → 执行导入 → 刷新列表 → 关闭弹窗
└─ 校验失败 → 显示错误信息 → 用户修正后重新上传
```

### 11.3 订单导出流程

```
用户点击导出按钮
    ↓
获取当前筛选条件
    ↓
调用导出API，生成Excel文件
    ↓
浏览器下载文件
    ↓
显示导出成功提示
```

### 11.4 批量派单流程（调度员）

```
用户勾选多个待派单订单（复选框）
    ↓
点击批量派单按钮
    ↓
跳转智能调度页面 /dispatch/center
    ↓
预选中已选订单，自动进入派单流程
```

## 15. 原型生成说明

### 15.1 页面布局建议

- **整体布局**：采用标准的列表页布局，顶部标题区 + 筛选区 + 表格区 + 分页区
- **筛选区布局**：采用横向排列，状态筛选和时间筛选使用控件组合，搜索框单独一行
- **表格布局**：固定列宽，订单号、客户信息、运输信息使用自适应宽度
- **操作按钮**：新建、导入、导出按钮位于页面右上角，使用主次按钮区分

### 15.2 关键交互优先级

1. **高优先级**：订单列表加载和展示、订单点击跳转详情、订单筛选功能
2. **中优先级**：订单导入/导出、批量派单跳转
3. **低优先级**：分页操作、每页条数切换

### 15.3 需要重点表达的业务逻辑

- 订单状态的视觉区分（使用不同颜色的状态标签）
- 筛选条件的即时反馈
- 空状态和加载状态的友好展示
- 导入/导出操作的用户引导

### 15.4 可使用模拟数据

需要准备以下模拟数据：

1. **订单列表数据**：至少20条订单，覆盖所有状态
   - 待派单：5条
   - 已派单：3条
   - 运输中：5条
   - 已到达：2条
   - 已签收：3条
   - 已完成：1条
   - 已取消：1条

2. **客户数据**：模拟不同客户名称（公司名称）

3. **地址数据**：模拟不同城市发货和收货地址

4. **货物数据**：模拟不同货物类型和重量

5. **承运数据**：部分订单有承运车辆和司机信息，部分为空

### 15.5 需要避免的原型误差

- 状态颜色必须与全局设计规范一致
- 筛选下拉框必须包含“全部“选项作为默认项
- 表格中的空值需要显示占位符（如“-”或”待派单”）
- 导入弹窗的文件上传区域需要有视觉反馈
- 批量选中时需要有明确的选中数量提示
- 分页组件需要展示总数据量和总页数
