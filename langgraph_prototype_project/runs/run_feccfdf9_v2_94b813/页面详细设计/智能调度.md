## 2. 页面目标与使用场景

### 2.1 页面目标

智能调度页面是物流系统的核心功能模块，旨在帮助调度员高效完成订单分配任务。页面统一管理待派单订单和可用资源，支持智能派单和手动派单两种模式，提供路线规划功能，最终实现快速准确的派单操作。

### 2.2 使用场景

| 场景 | 场景描述 | 触发条件 |
| --- | --- | --- |
| 场景1 | 调度员查看待派单订单，了解当前积压订单情况 | 页面加载 |
| 场景2 | 使用智能派单功能，系统自动生成最优派单方案 | 点击"智能派单"按钮 |
| 场景3 | 手动选择车辆和司机进行派单 | 点击订单的"派单"按钮 |
| 场景4 | 查看推荐路线的详细信息 | 点击订单的"查看路线"按钮 |
| 场景5 | 确认派单并通知司机 | 点击"确认派单"按钮 |
| 场景6 | 撤销已派单但未执行的订单 | 点击"撤销"按钮 |
| 场景7 | 更换订单的车辆和司机 | 点击"重新派单"按钮 |

## 7. 业务状态与状态流转

### 7.1 订单状态

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |
| --- | --- | --- | --- | --- | --- | --- |
| 待派单 | PENDING | 订单已创建，等待派单 | 新建订单完成 | 智能派单、手动派单 | ASSIGNED | 无可用资源提示 |
| 已派单 | ASSIGNED | 订单已分配给车辆和司机 | 派单确认完成 | 查看详情、撤销 | IN_TRANSPORT | 撤销后可恢复为PENDING |
| 运输中 | IN_TRANSPORT | 车辆正在运输 | 司机接单开始运输 | 查看轨迹、异常上报 | ARRIVED | 异常时跳转到异常处理 |
| 已到达 | ARRIVED | 车辆已到达目的地 | 司机到达目的地 | 签收确认 | SIGNED | - |
| 已签收 | SIGNED | 客户已签收 | 客户签收完成 | 查看详情 | COMPLETED | - |
| 已完成 | COMPLETED | 订单全部完成 | 签收确认完成 | 查看详情 | - | - |
| 已取消 | CANCELLED | 订单已取消 | 调度员取消订单 | 查看详情 | - | - |

### 7.4 状态流转图

```
PENDING(待派单)
    ↓ [智能派单/手动派单]
ASSIGNED(已派单)
    ↓ [司机接单]
IN_TRANSPORT(运输中)
    ↓ [到达目的地]
ARRIVED(已到达)
    ↓ [客户签收]
SIGNED(已签收)
    ↓ [系统处理]
COMPLETED(已完成)

PENDING(待派单) → [取消订单] → CANCELLED(已取消)
ASSIGNED(已派单) → [撤销派单] → PENDING(待派单)
IN_TRANSPORT(运输中) → [异常发生] → 异常处理流程
```

## 9. 交互逻辑

### 9.1 页面初始化

* 默认展示：加载待派单订单列表、可用车辆列表、可用司机列表
* 默认排序：按订单创建时间升序排列
* 默认筛选：只显示PENDING状态的订单
* 初始化异常：如果数据加载失败，展示错误提示和重试按钮

### 9.2 智能派单流程

```
1. 调度员在待派单订单列表中选中一个或多个订单
2. 点击"智能派单"按钮
3. 系统根据订单货物重量、目的地、期望时间等因素
4. 结合可用车辆的载重、位置、状态
5. 结合可用司机的状态、准驾车型、位置
6. 生成最优派单方案（模拟智能算法）
7. 展示推荐方案：
   - 展示每个订单推荐的车辆和司机
   - 展示预计路线距离和时间
   - 展示匹配度评分
8. 调度员可调整推荐方案
9. 点击"确认派单"按钮
10. 系统执行派单操作：
    - 更新订单状态为已派单
    - 更新车辆状态为使用中
    - 更新司机状态为工作中
    - 创建运输任务
    - 弹出成功提示
    - 通知司机（模拟弹窗提示）
```

### 9.3 手动派单流程

```
1. 调度员点击订单卡片的"派单"按钮
2. 弹出派单弹窗：
   - 左侧：订单信息
   - 中间：可选车辆列表（筛选可用车辆）
   - 右侧：可选司机列表（筛选空闲司机）
3. 调度员选择车辆
4. 调度员选择司机
5. 系统自动计算路线规划
6. 展示路线详情
7. 点击"确认派单"按钮
8. 系统执行派单操作（与智能派单第10步相同）
```

### 9.4 撤销派单流程

```
1. 调度员找到已派单订单
2. 点击"撤销"按钮
3. 弹出确认框："确认撤销该订单的派单？撤销后订单将恢复为待派单状态"
4. 点击"确认撤销"
5. 系统执行撤销操作：
   - 更新订单状态为待派单
   - 更新车辆状态为空闲
   - 更新司机状态为空闲
   - 删除运输任务
6. 弹出成功提示
```

### 9.5 路线规划展示

```
1. 调度员点击订单的"查看路线"按钮
2. 弹出路线规划弹窗：
   - 地图展示起点和终点
   - 绘制推荐路线
   - 显示途经点（如有）
   - 显示预计距离：XXX公里
   - 显示预计时间：X小时X分钟
3. 调度员查看后点击关闭
```

## 12. 异常、空状态和边界场景

| 场景 | 触发条件 | 页面表现 | 用户操作 | 恢复方式 |
| --- | --- | --- | --- | --- |

## 15. 原型生成说明

### 15.1 页面布局建议

* 整体采用左右分栏布局，左侧宽度60%，右侧宽度40%
* 左侧：待派单订单列表，采用卡片式设计
* 右上：资源面板，展示可用车辆和司机统计卡片及列表
* 右下：派单操作面板，选中订单和资源后显示
* 配色：主色调蓝色 #1890ff，卡片白色背景，间隔灰 #f0f0f0

### 15.2 关键交互优先级

1. **智能派单功能**（最高优先级）：展示系统自动生成的推荐方案，包含匹配度评分
2. **手动派单功能**（高优先级）：支持选择车辆和司机，展示路线规划
3. **路线规划展示**（中优先级）：使用地图展示路线，显示距离和时间
4. **资源状态展示**（中优先级）：实时展示可用资源数量和状态

### 15.3 需要重点表达的业务逻辑

* 智能派单算法：根据货物重量匹配载重，根据目的地匹配位置最近的车辆和司机
* 派单校验：检查车辆载重、司机准驾车型、资源可用状态
* 状态联动：派单后自动更新订单、车辆、司机的状态
* 派单通知：模拟司机端收到派单通知的弹窗效果

### 15.4 可使用模拟数据

* 准备10条待派单订单，涵盖不同货物类型、重量、路线
* 准备8辆可用车辆，涵盖不同车型、载重、车牌号
* 准备10名可用司机，涵盖不同准驾车型、姓名
* 订单数据包含：北京→上海、广州→深圳、武汉→杭州等路线
* 货物类型包含：电子产品、服装、食品、建材、机械设备等

### 15.5 需要避免的原型误差

* 智能派单不能只是简单随机分配，需要展示推荐逻辑（如：优先选择载重匹配、距离最近的）
* 路线规划需要真实展示地图效果（可使用静态地图图片或简化版地图组件）
* 派单成功后需要展示状态变化和通知效果，不能只是提示
* 资源列表需要展示状态筛选功能，点击可选中
* 批量派单需要展示多个订单同时处理的场景
