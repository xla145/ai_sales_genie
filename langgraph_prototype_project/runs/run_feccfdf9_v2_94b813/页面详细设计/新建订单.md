## 2. 页面目标与使用场景

- **页面目标**：
  - 支持运营人员快速录入新的物流订单
  - 提供完整的订单信息填写界面，包含发货信息、收货信息、货物信息、时间要求等
  - 实现表单校验和数据保存

- **使用场景**：
  - 客户通过电话或线下下单，运营人员手动创建订单
  - 调度员或客服从其他渠道获取订单信息后录入系统
  - 批量导入订单后的单条补充录入

- **触发入口**：
  - 订单列表页点击「新建订单」按钮
  - 首页快捷入口点击「新建订单」
  - 其他需要创建订单的业务场景

- **输出结果**：
  - 新订单创建成功，状态为「待派单(PENDING)」
  - 自动生成订单编号
  - 跳转到订单列表页或订单详情页

## 7. 业务状态与状态流转

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |

## 9. 交互逻辑

### 9.1 页面初始化

- **默认状态**：
  - 订单编号自动生成（格式：ORD + 年月日 + 6位序号，如 ORD20241201000001）
  - 所有可输入字段为空
  - 期望送达时间为空，可选择
  - 状态显示为「待派单」（PENDING）

- **初始化加载**：
  - 加载货物类型枚举数据
  - 加载特殊说明枚举数据
  - 加载行政区划数据（省市县三级联动）

### 9.2 表单填写与校验

- **实时校验**：
  - 失去焦点时触发当前字段校验
  - 必填字段未填写时显示红色错误提示「请输入/选择XXX」
  - 格式不正确时显示具体错误提示（如手机号格式错误）

- **提交前校验**：
  - 所有必填字段不能为空
  - 手机号必须为11位数字
  - 期望送达时间必须大于当前时间
  - 最迟送达时间（如果填写）必须大于期望送达时间
  - 货物数量、重量必须大于0

- **校验规则表**：
  | 字段 | 校验规则 | 错误提示 |
  |------|----------|----------|
  | 客户名称 | 必填，最长50字符 | 请输入客户名称 |
  | 客户电话 | 必填，11位手机号 | 请输入正确的手机号 |
  | 发货人 | 必填，最长30字符 | 请输入发货人 |
  | 发货人电话 | 必填，11位手机号 | 请输入正确的手机号 |
  | 发货省市区 | 必填 | 请选择发货省市区 |
  | 发货详细地址 | 必填，最长100字符 | 请输入详细地址 |
  | 收货人 | 必填，最长30字符 | 请输入收货人 |
  | 收货人电话 | 必填，11位手机号 | 请输入正确的手机号 |
  | 收货省市区 | 必填 | 请选择收货省市区 |
  | 收货详细地址 | 必填，最长100字符 | 请输入详细地址 |
  | 货物名称 | 必填，最长50字符 | 请输入货物名称 |
  | 货物类型 | 必填 | 请选择货物类型 |
  | 货物数量 | 必填，正整数≥1 | 请输入正确的货物数量 |
  | 货物重量 | 必填，正数 | 请输入正确的货物重量 |
  | 期望送达时间 | 必填，大于当前时间 | 请选择期望送达时间，且必须大于当前时间 |
  | 最迟送达时间 | 大于期望送达时间 | 最迟送达时间必须大于期望送达时间 |

### 9.3 客户名称联想

- **功能说明**：
  - 用户输入客户名称时，支持模糊搜索历史客户
  - 下拉显示匹配的历史客户列表
  - 选择历史客户后自动填充客户电话

- **触发条件**：
  - 客户名称输入框获得焦点后，已有空内容时显示历史记录
  - 输入字符后显示匹配结果

### 9.4 省市区联动

- **功能说明**：
  - 选择发货省市区时，三级联动
  - 选择收货省市区时，三级联动
  - 选择发货省后，城市列表更新；选择城市后，区县列表更新

### 9.5 时间选择控制

- **期望提货时间**：
  - 可选择当前时间之后的时间
  - 默认不限制最早时间

- **期望送达时间**：
  - 必填
  - 必须大于当前时间
  - 如果填写了期望提货时间，则必须大于期望提货时间

- **最迟送达时间**：
  - 选填
  - 如果填写，必须大于期望送达时间

### 9.6 表单提交

- **提交流程**：
  1. 用户点击「提交订单」按钮
  2. 系统执行全量表单校验
  3. 校验失败时，滚动到第一个错误位置，错误字段下方显示错误提示
  4. 校验通过时，显示加载状态
  5. 提交成功后，显示成功提示toast，1.5秒后跳转到订单列表页
  6. 提交失败时，显示错误提示toast，保留表单数据

### 9.7 取消/返回处理

- **触发条件**：
  - 点击「取消」按钮
  - 点击「返回」按钮
  - 浏览器返回按钮

- **判断逻辑**：
  - 如果表单内容发生变化（任意字段有值），弹窗确认
  - 如果表单内容未发生变化，直接跳转/关闭

- **弹窗内容**：
  - 标题：「确认离开」
  - 内容：「确定要离开吗？已填写的内容将不会保存。」
  - 按钮：「取消」（关闭弹窗）、「确定」（离开页面）

### 10.2 分支流程

#### 分支1：客户名称联想
- **触发**：客户名称输入框输入内容
- **流程**：
  1. 显示匹配的历史客户下拉列表
  2. 用户选择或继续输入
  3. 选择历史客户时自动填充电话
- **输出**：填充客户信息

#### 分支2：校验失败
- **触发**：提交订单时必填字段为空或格式错误
- **流程**：
  1. 系统定位到第一个错误字段
  2. 滚动页面使错误字段可见
  3. 错误字段下方显示错误提示
  4. 用户修正错误后重新提交
- **输出**：错误提示展示

#### 分支3：取消创建
- **触发**：点击取消或返回按钮
- **流程**：
  1. 弹窗确认是否离开
  2. 用户确认后返回订单列表
  3. 用户取消则继续填写
- **输出**：页面跳转或继续当前操作

## 14. 原型生成说明

- **页面布局建议**：
  - 采用紧凑的纵向布局，从上到下依次展示各信息组
  - 每个信息组使用卡片样式区分，带有分组标题
  - 表单字段采用两列或三列排列，提高空间利用率
  - 底部提交按钮固定在页面底部或跟随表单内容

- **关键交互优先级**：
  1. 省市区三级联动选择（高）
  2. 表单实时校验和提交校验（高）
  3. 客户名称联想功能（中）
  4. 时间选择逻辑控制（中）
  5. 取消/返回确认弹窗（低）

- **需要重点表达的业务逻辑**：
  - 订单状态创建后默认为「待派单(PENDING)」
  - 订单编号自动生成规则
  - 发货地和收货地的省市区联动选择
  - 期望送达时间的必填和校验逻辑
  - 货物信息的必填字段校验

- **可使用模拟数据**：
  - 历史客户名称和电话联想（模拟10-20条数据）
  - 行政区划数据（全国省份、城市、区县）
  - 货物类型枚举
  - 特殊说明枚举

- **需要避免的原型误差**：
  - 表单校验必须实时反馈，不能等到提交才发现错误
  - 省市选择必须三级联动，不能只选省份
  - 时间选择必须带时分秒
  - 提交按钮必须在所有必填项填写后才能点击（可使用禁用状态）
  - 订单编号必须是自动生成且只读
