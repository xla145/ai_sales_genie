|------|------|

## 2. 页面目标与使用场景

### 2.1 页面目标

1. 展示订单完整信息，包括基本信息、货物信息、运输信息、关联资源等
2. 实时展示订单运输状态和轨迹
3. 支持订单签收确认操作
4. 支持订单取消和修改操作（限特定状态）
5. 展示订单相关操作记录和处理日志

### 2.2 使用场景

| 场景 | 触发条件 | 用户行为 | 输出结果 |
|------|----------|----------|----------|
| 查看订单详情 | 从订单列表点击订单 | 查看完整订单信息 | 展示订单详情页面 |
| 订单状态跟踪 | 订单处于运输中状态 | 查看实时运输状态和轨迹 | 展示轨迹地图和状态信息 |
| 签收确认 | 订单到达目的地 | 点击签收按钮，上传回单 | 订单状态变更为已签收 |
| 订单取消 | 订单未开始执行 | 点击取消按钮，填写原因 | 订单状态变更为已取消 |
| 订单修改 | 订单为待派单状态 | 点击修改按钮 | 跳转到订单编辑页面 |

### 2.3 触发入口

1. 订单列表页点击订单记录
2. 首页待办事项点击订单
3. 智能调度页点击订单查看详情
4. 运输轨迹页点击车辆关联订单

## 7. 业务状态与状态流转

### 7.1 订单状态

| 状态名称 | 状态编码 | 状态说明 | 触发条件 | 可执行操作 | 下一状态 | 异常处理 |
|----------|----------|----------|----------|------------|----------|----------|
| 待派单 | PENDING | 订单已创建，等待派单 | 新建订单完成 | 智能派单、手动派单、修改、取消 | ASSIGNED | - |
| 已派单 | ASSIGNED | 订单已分配车辆和司机 | 派单确认完成 | 开始运输、取消、重新派单 | IN_TRANSPORT | 车辆或司机异常可重新派单 |
| 运输中 | IN_TRANSPORT | 车辆正在运输 | 司机确认开始运输 | 到达确认、异常上报 | ARRIVED | 可上报异常 |
| 已到达 | ARRIVED | 车辆已到达目的地 | 司机确认到达 | 签收确认 | SIGNED | - |
| 已签收 | SIGNED | 客户已签收 | 客户签收确认 | 确认完成 | COMPLETED | 签收异常可处理 |
| 已完成 | COMPLETED | 订单全部完成 | 签收确认完成 | 查看、导出 | - | - |
| 已取消 | CANCELLED | 订单已取消 | 取消操作确认 | 查看 | - | 可重新下单 |

### 7.2 状态流转图

```
PENDING → ASSIGNED → IN_TRANSPORT → ARRIVED → SIGNED → COMPLETED
   |          |            |            |         |
   +----取消----→ CANCELLED  +---异常--→ 异常处理
                           |
                      +----异常----→ 异常处理 → 继续/终止
```

### 7.3 操作与状态对应关系

| 操作 | 可执行状态 | 操作后状态 | 权限角色 |
|------|------------|------------|----------|
| 智能派单 | PENDING | ASSIGNED | 调度员 |
| 手动派单 | PENDING | ASSIGNED | 调度员 |
| 重新派单 | ASSIGNED | ASSIGNED | 调度员 |
| 开始运输 | ASSIGNED | IN_TRANSPORT | 司机 |
| 到达确认 | IN_TRANSPORT | ARRIVED | 司机 |
| 签收确认 | ARRIVED | SIGNED | 运营人员 |
| 确认完成 | SIGNED | COMPLETED | 运营人员 |
| 取消订单 | PENDING、ASSIGNED | CANCELLED | 运营人员 |
| 修改订单 | PENDING | PENDING | 运营人员 |

## 9. 交互逻辑

### 9.1 页面初始化

1. **数据加载**：页面加载时，根据路由参数orderId获取订单详情数据
2. **状态判断**：根据订单状态判断可显示的Tab和可执行的操作按钮
3. **权限匹配**：根据当前用户角色过滤可执行操作
4. **默认Tab**：默认显示"基本信息"Tab

### 9.2 Tab切换逻辑

| Tab名称 | 切换条件 | 内容说明 |
|---------|----------|----------|
| 基本信息 | 始终可切换 | 展示订单基本信息、货物信息、财务信息 |
| 运输信息 | 订单状态为ASSIGNED及之后 | 展示车辆信息、司机信息、路线信息 |
| 轨迹信息 | 订单状态为IN_TRANSPORT及之后 | 展示轨迹地图和时间轴 |
| 操作日志 | 始终可切换 | 展示订单所有操作记录 |

### 9.3 派单操作流程

```
点击派单按钮
    ↓
判断派单方式（智能/手动）
    ↓
智能派单：系统计算推荐方案 → 显示推荐结果 → 确认派单
手动派单：选择车辆 → 选择司机 → 确认派单
    ↓
调用派单接口
    ↓
成功：更新订单状态 → 显示成功提示 → 刷新页面
失败：显示错误提示 → 保持当前状态
```

### 9.4 签收操作流程

```
点击签收按钮
    ↓
打开签收弹窗
    ↓
填写签收信息（签收人、签收备注）
```

## 15. 原型生成说明

### 15.1 页面布局建议

1. **整体布局**：采用左右布局，左侧为订单概览信息（可折叠），右侧为详情内容区
2. **顶部区域**：固定高度80px，包含订单编号、状态标签、操作按钮组
3. **Tab导航**：固定高度48px，4个Tab项，等宽分布
4. **内容区域**：自适应高度，最大高度时出现滚动条
5. **底部区域**：固定高度60px，包含返回按钮

### 15.2 关键交互优先级

1. **高优先级**：状态展示、操作按钮响应、Tab切换
2. **中优先级**：轨迹地图交互、签收弹窗、取消确认框
3. **低优先级**：操作日志分页、轨迹时间轴滚动

### 15.3 需要重点表达的业务逻辑

1. **订单状态流转**：清晰展示当前状态和可执行操作
2. **派单流程**：智能派单和手动派单的完整交互流程
3. **签收流程**：签收信息填写和图片上传的完整交互
4. **轨迹展示**：地图上的轨迹线和关键节点标记
5. **操作日志**：订单完整操作历史的可追溯性
