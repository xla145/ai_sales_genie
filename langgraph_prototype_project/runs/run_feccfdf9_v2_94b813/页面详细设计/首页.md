## 2. 页面目标与使用场景

### 2.1 页面目标

1. 展示今日核心运营指标（订单数、运输中订单、异常订单、收入等）
2. 展示待处理任务（待派单订单、待审批异常、待办事项）
3. 提供常用功能快捷入口（新建订单、车辆管理、调度中心等）
4. 展示关键指标趋势图表

### 2.2 使用场景

| 场景 | 描述 | 触发条件 |
|------|------|----------|
| 日常工作查看 | 用户登录系统后首先查看首页，了解当日运营概况 | 用户登录 |
| 待办任务处理 | 查看待办事项列表，快速进入处理页面 | 存在待办任务 |
| 快速导航 | 通过快捷入口快速进入常用功能页面 | 用户需要执行特定操作 |
| 数据趋势分析 | 查看关键指标趋势图表 | 管理人员需要了解运营趋势 |

### 2.3 触发入口

1. 用户登录系统后自动跳转至首页
2. 点击顶部导航栏"首页"菜单

### 2.4 输出结果

用户获取当日运营概况，了解待办任务，快速进入目标功能页面

## 9. 交互逻辑

### 9.1 页面初始化

1. **默认展示内容**：
   - 今日概览4个指标卡片（订单数、运输中、异常订单、今日收入）
   - 待办事项列表（按优先级排序）
   - 快捷入口（根据用户角色显示可用入口）
   - 趋势图表（默认显示近7天数据）

2. **默认筛选**：无

3. **初始化异常处理**：
   - 数据加载失败：显示加载失败提示，提供重试按钮
   - 网络异常：显示网络异常提示，2秒后自动重试

### 9.2 指标卡片交互

| 交互 | 行为 |
|------|------|
| 悬停 | 卡片轻微上浮，添加阴影（transform: translateY(-2px), box-shadow增强） |
| 点击 | 跳转到对应列表页（订单数→订单列表，运输中→订单列表筛选运输中，异常订单→异常处理，今日收入→数据看板） |

### 9.3 待办事项交互

| 交互 | 行为 |
|------|------|
| 列表项悬停 | 背景色变为#f5f7fa |
| 点击待办项 | 跳转到对应的处理页面 |
| 点击"查看全部" | 跳转到对应的列表页面，显示全部待办 |

### 9.4 快捷入口交互

| 交互 | 行为 |
|------|------|
| 图标悬停 | 图标放大1.1倍，颜色加深，显示文字提示 |
| 点击 | 跳转到目标功能页面 |
| 无权限入口 | 不显示或显示为禁用状态 |

### 9.5 趋势图表交互

| 交互 | 行为 |
|------|------|
| 时间切换 | 点击近7天/近30天按钮，图表数据切换，激活按钮高亮 |
| 图表悬停 | 显示数据点的详细数值tooltip |
| 图表点击 | 无特殊行为 |

### 9.6 用户菜单交互

| 交互 | 行为 |
|------|------|
| 点击头像/用户名 | 展开下拉菜单，包含：个人中心、修改密码、退出登录 |
| 点击退出登录 | 弹出确认框，确认后退出到登录页 |
| 点击菜单外部 | 下拉菜单收起 |

## 12. 异常、空状态和边界场景

| 场景 | 触发条件 | 页面表现 | 用户操作 | 恢复方式 |
|------|----------|----------|----------|----------|

## 15. 原型生成说明

### 15.1 页面布局建议

1. **整体布局**：采用左侧固定菜单+右侧内容区的经典后台布局
2. **顶部导航**：Logo在左，用户信息在右，高度60px
3. **左侧菜单**：宽度240px，可折叠，收缩后宽度64px
4. **主内容区**：左右 padding 24px，上下 padding 20px
5. **卡片区域**：使用 Card 组件，border-radius: 8px，轻微阴影

### 15.2 关键交互优先级

1. **高优先级**：指标卡片点击跳转、待办事项点击跳转、快捷入口点击跳转
2. **中优先级**：趋势图表时间切换、用户菜单展开
3. **低优先级**：卡片悬停效果、图标悬停效果

### 15.3 需要重点表达的业务逻辑

1. 不同角色登录后看到的快捷入口不同（根据权限过滤）
2. 待办事项数量实时更新，点击可快速处理
3. 指标卡片数据来自实时统计，点击可查看详情列表
4. 趋势图表支持时间范围切换

### 15.5 需要避免的原型误差

1. **不要使用静态图片**：所有图表必须可交互，悬停显示数据点
2. **不要忽略权限控制**：不同角色看到的快捷入口必须不同
3. **不要遗漏加载状态**：数据加载必须显示骨架屏或加载动画
4. **不要忽略空状态**：待办事项为0时要显示"暂无待办"提示
5. **保持颜色一致**：严格按照状态颜色规则（蓝色-进行中、绿色-完成、红色-异常、橙色-待处理、灰色-取消/停用）

### 15.6 原型文件要求

1. 使用Vue3 + Element Plus或React + Ant Design Vue实现
2. 地图组件使用ECharts或其他图表库
3. 响应式布局，最小宽度1200px
4. 所有点击操作必须有响应（跳转、提示、加载等）
5. 准备充足的模拟数据确保演示效果
